Competitor Map

用户自带 Bot 的游戏世界，竞品其实比看起来多。

这份页面把 ClawGame 所处赛道拆成直接竞品、邻近竞品、平台型竞品和新一代垂直 bot 平台四层。核心结论不是“市场上没人做”，而是“真正把持续 RPG 世界、bot 角色成长、公开观战和赛季竞争整合起来的人还很少”。

结论摘要

先回答最重要的问题：这个方向并非空白，但完全等同于 ClawGame 的成熟产品仍然不多。

截至 2026-03-26 的公开资料
市场状态

“提交 bot、自动对战、看榜单”的平台已经很多；“程序驱动持久世界”的代表也存在；“人类围观、agent 参赛”的表达在 2025-2026 年明显增多。

机会所在

把 用户自带 bot + RPG 世界 + live service + 公开观战 + 周期赛事 真正组合好的产品，仍然很少。

商业方向

竞品共同证明了三种可行路径：月卡/订阅、能力与额度收费、赛事与组织者销售。

Screeps

最接近 ClawGame 长期愿景的成熟产品。也是你做商业模式和产品表达时最该长期跟踪的对象。

直接竞品

产品形态

官方将产品拆成 `World` 和 `Arena` 两层：前者是持续开放世界 MMO，后者是异步匹配竞技场。

  • `World` 强调 persistent open world、真实 JavaScript、24/7 持续运行。
  • `Arena` 强调 rating-based matchmaking、seasons、asynchronous PvP。
  • 这一点直接证明“程序驱动世界 + 持续竞争”是可以成立的。

商业模式

公开页面显示它是混合收费，而不是单一订阅。

  • Steam 买断进入。
  • World 商店包含 CPU unlock、access keys、pixels 等。
  • 能力解锁和持续运行资源可以付费购买。
  • 有赛季和 creator program，说明内容分发也被纳入商业体系。

对 ClawGame 的启发

  • 月卡制可以成立，但更好的表达是“运行能力”而不是“入场门票”。
  • 可以把配额、运行额度、回放、赛季资格做成付费层。
  • 基础接入和能力解锁分层，是很值得参考的设计。

对你的威胁与差异化

  • 它已经占据“程序员世界游戏”的认知高地。
  • 你不能做成另一个通用 RTS 编程世界。
  • 你的差异化应放在 RPG 世界、任务/地下城/竞技场、公开观战站和 bot 角色故事化上。

Botzone

不是持久世界产品，但它非常能说明“提交 bot、打比赛、看排行榜”这一需求长期存在。

邻近竞品

产品形态

  • 多游戏 bot 比赛平台，覆盖棋牌和策略类项目。
  • 提供 bot 提交、游戏桌对局、排行榜、本地 AI 连接、比赛规则。
  • 公开页面显示承办正式 AI 竞赛，并提供样例程序、协议与裁判程序。

模式判断

  • 更像竞赛平台与教育/学术平台的结合。
  • 截至目前未在公开页面看到清晰的个人订阅价格。
  • 比赛组织能力本身就是平台价值的一部分。

对 ClawGame 的启发

  • 必须尽早提供样例 bot、调试方式和清晰协议。
  • 没有赛事的 bot 平台很难持续活跃。
  • 组织比赛、维护规则、做榜单，本身就能构成商业价值。

局限提醒

  • Botzone 不强调世界叙事和 live service。
  • ClawGame 不该退化成“另一个提交 bot 的平台外壳”。

BotClash

新兴 bot-only 竞技场。它最大的参考价值不是收入，而是产品叙事和接入路径。

直接竞品

产品形态

  • 官网直接写 `AI Battle Arena`、`Bots Only`、`BETA v0.1`。
  • 用户注册 agent，获取 agentId 与 apiKey。
  • REST 进入匹配，WebSocket 进入对局。
  • 前台强调直播、排行榜、战斗日志和 bot 聊天。

模式判断

  • 当前更像社区冷启动阶段，重点放在活跃和围观感上。
  • 公开页面暂未见到明确的收费页。
  • 表达上非常 agent-native，和你的“人类围观、bot 参赛”方向高度接近。

对 ClawGame 的启发

  • 官网不该像后台，而应像 bot 世界观测站。
  • 接入流程必须短，从注册到开始运行 bot 的路径要尽量直接。
  • 战报、排行榜、热点 bot 和局部事件是很重要的前台资产。

差异化机会

  • BotClash 更像快节奏 1v1 arena。
  • ClawGame 应坚持长期世界、多系统循环与角色成长，避免做成它的慢版本。

Elympics

不一定抢同一批终端用户，但很像你未来若走 B 端平台化时会遇到的参照物和潜在竞争者。

平台型竞品

产品形态

  • 公开产品页强调 Bot Deployment、Smart Matchmaking、Verifiable Replays、Leaderboards、Tournaments & Duels。
  • bot 可自动填充空位、降低等待时间、用于测试游戏。
  • 回放会自动存储，可用于直播与 debug。

模式判断

  • 这是典型的 B2B SaaS / infra 思路。
  • 卖的是托管、匹配、回放和比赛基础设施能力，而不是单个 bot 用户月卡。

对 ClawGame 的启发

  • 今天做的回放、赛季配置、私有联赛、规则版本管理，未来都可能抽象成平台能力。
  • 这说明 ClawGame 有机会不只是游戏，也能成长为平台产品。

与你的差异

  • Elympics 强在通用平台，弱在内容化世界表达。
  • ClawGame 强在 bot-first RPG 世界与公开叙事，这仍是短期差异化空间。

CodeCombat AI League

玩法不是最像你的，但它非常值得参考“组织者销售”和“赛事包装”的方式。

邻近竞品

产品形态

  • 官方将 AI League 描述为 academic esports。
  • 支持学校、班级、组织层面的竞赛包装。
  • Premium 提供私有 clan、组内 leaderboard 等能力。

模式判断

  • 公开信息显示其收入结构包含 Premium、教育销售和组织化赛事方案。
  • 说明排行榜、私有群组和赛事都可以变成付费权益。

对 ClawGame 的启发

  • 组织者版很可能比纯个人月卡更早带来收入。
  • 私有联赛、团队榜单和组内回放适合视为单独付费能力。

适用边界

  • 它更偏教育和品牌化包装，不是持久世界。
  • 你可以借鉴销售结构，但不必模仿玩法。

CodinGame

它说明“大规模 bot 编程社区 + 多活动轮换”是能长期维持的，这对你做赛季和特别赛很有帮助。

邻近竞品

产品形态

  • 有成熟的 bot programming 赛道，覆盖 multi-agent、resource management、pathfinding、simulation 等多类题型。
  • 支持公开活动和私有活动。
  • 新提交会触发重新匹配，形成持续榜单竞争。

模式判断

  • bot 编程更像其开发者生态和品牌流量入口。
  • 商业价值更可能来自平台整体、企业合作和赞助赛事。

对 ClawGame 的启发

  • 长期世界榜之外，可以增加限时特别赛、主题赛和短期挑战榜。
  • 多活动层能帮助避免世界玩法过快收敛。

与你的差异

  • CodinGame 强在多题型与大社区。
  • ClawGame 应该更像一个持续 live service 世界，而不是一组离散竞赛题目。

Battlecode

经典学术竞赛先例。它告诉你高门槛 bot 竞赛也可以形成长期社群，只是商业化路径不同。

邻近竞品

产品形态

  • 官方 About 页将其定义为 real-time strategy game 竞赛。
  • 参赛者编写 autonomous player。
  • 一个赛季内有 scrimmages、tournaments、总决赛和奖励。

模式判断

  • 更像学术/社群竞赛，而不是持续运营的商业游戏平台。
  • 但其赛季节奏和总决赛层次非常值得借鉴。

对 ClawGame 的启发

  • 赛季不一定全年无休，也可以有冲刺、资格赛、总决赛层次。
  • 强竞赛用户会非常在意算力限制、公平性和规则稳定性。

适用边界

  • Battlecode 不是持续世界产品。
  • ClawGame 需要吸收其竞赛强度，但保持 live service 的持续性。

Robocode / Tank Royale

历史很长的 bot 战斗传统。它更像形态先例，而不是直接商业竞品。

形态先例

产品形态

  • 核心是编写机器人自动对战。
  • Tank Royale 文档强调多语言、Viewer、Recorder、Replay、Debugging、Team Messages。
  • 还专门提到适合比赛展示和大屏观战。

模式判断

  • 偏开源社区和长期知识积累，不是强商业化产品。
  • 但它证明“写 bot 看对战”本身有很长的生命周期。

对 ClawGame 的启发

  • Replay、viewer 和战术分析不是附属功能，而是核心资产。
  • 长期社群会自然产生策略文章、复盘内容和 bot 流派。

对你的提醒

  • 如果想经营长期社区，必须给高手足够深的优化空间。
  • 否则只会停留在“新鲜一阵”的体验层。

新一代垂直 Bot 平台

这些产品很新，但它们集中体现了 2025-2026 年 agent-native 产品的表达方式。

实验新秀

ChessArena

  • 官网强调 5 分钟部署、真实 ELO、sandboxed environment、same hardware same limits。
  • 说明 builder-first 产品可以把公平硬件和代码所有权放在前台。
  • 对你的启发是：bot 注册和开打路径可以非常短，公平约束应该明确写给用户看。

BotArena

  • 公开页面强调 transparent match logs、deterministic replay、team workspaces、versioned deployments。
  • 这很像 bot 产品经理工具和竞技场的结合。
  • 对你的启发是：版本管理、A/B 对战、团队空间都可能成为高价值付费功能。

Arena Protocol

  • 直接写出 `AI Agents Compete. Humans Spectate.`
  • 支持多游戏，自动匹配、自动记录、ELO 排名、Replay API。
  • 这是目前和你“人类围观、agent 才是玩家”叙事最接近的一类产品。

Constrained Agent Arena

  • 强调 `Compete with Code, Not Clicks`、No GPUs、No massive models、Elo-style ratings、match replays。
  • 代表“受限算力、公平约束、研究友好”的 agent 平台方向。
  • 对你的启发是：如果未来允许 LLM/agent 深度参与，必须尽早定义算力、延迟和公平边界。

对 ClawGame 的启发

看完竞品之后，真正值得记住的不是“谁做过”，而是“哪些模式已经被不同人分别验证过”。

定位

你不能只做“又一个 bot 平台”。最值得坚持的是 RPG 世界、任务与地下城、公开世界动态、角色成长和赛季竞技场。

收费

最合理的路径依然是 个人月卡 + 能力/额度增购 + 团队/组织者套餐 + 未来 B 端授权。

优先开发

商业化前最该补的是 bot 接入、认证、配额、日志、回放、赛季配置、私有联赛和公开站叙事能力。

最该长期盯的对象

  • Screeps：看持久世界和收费逻辑。
  • BotClash / Arena Protocol：看 AI agents compete, humans spectate 的产品表达。
  • CodinGame：看多活动和长期社区运营。
  • ChessArena / BotArena：看 builder-first、bot 托管即产品的做法。
  • Elympics：看未来平台化的 SaaS 方向。

最后的战略判断

  • 不要做成 Screeps 的低配替代。
  • 不要做成 Botzone 的另一种壳。
  • 不要做成 BotClash 的慢版本。
  • 也不要做成 CodinGame 上一题更重的长期题。
  • 你应该被理解为：一个让用户自带 bot，在持续运行的 RPG 世界里成长、竞争、被围观，并最终形成赛季与联赛生态的平台。
