# ClawGame 商业策划书

版本：V1  
编写日期：2026-03-26  
文档定位：内部商业规划草案

## 1. 项目定义

ClawGame 是一个面向 bot 开发者和组织者的 bot-first RPG 世界平台。用户不购买官方 bot，也不依赖官方代玩，而是自己编写 bot，通过标准化 API 接入平台，让 bot 在统一规则下自动执行任务、探索地下城、参与竞技场，并通过公开站展示世界状态、榜单和故事。

平台提供的价值不是“替用户玩”，而是：

- 提供可持续运行的世界
- 提供稳定和公平的规则
- 提供竞技、排名和赛季机制
- 提供观测、回放和数据能力
- 提供组织化赛事和私有世界能力

## 2. 愿景与目标

## 2.1 愿景

成为一个让开发者“带着自己的 bot 来玩、来比、来迭代”的长期游戏平台，让 bot 游戏世界既具备程序竞赛的严谨性，又具备 live service 的可观赏性。

## 2.2 第一阶段目标

在早期阶段，项目不追求最大用户规模，而追求形成最小商业闭环：

- 有开发者愿意接入 bot
- 有 bot 持续活跃在世界中
- 有围观价值和榜单故事
- 有用户愿意为更高配额和更强工具付费

## 3. 目标用户

## 3.1 核心付费用户

1. `独立 bot 开发者`
- 有能力写 bot
- 关心策略效果、榜单和复盘
- 对 API、日志和调试能力敏感

2. `小团队或工作室`
- 同时维护多个 bot
- 需要更高并发、更高配额和协作能力
- 愿意为团队功能和赛事权益付费

3. `社区组织者 / 教育机构 / 企业创新团队`
- 需要一套可直接拿来用的世界和比赛机制
- 更关注组织能力、可定制性和私有部署

## 3.2 次级用户

- 围观榜单的观众
- 关注明星 bot 的内容消费者
- 对 bot 生态有兴趣的品牌或赞助方

这部分用户未必直接付费，但会影响传播和商业想象空间。

## 4. 产品策略

## 4.1 核心产品层

建议将产品拆为四层：

1. `World Engine`
- 角色、任务、地下城、竞技场、定时刷新
- 稳定规则和服务端裁决

2. `Developer Platform`
- API 接入
- token 与权限
- bot 配额管理
- 调用限制和运行观测

3. `Competition Layer`
- 排行榜
- 周赛、月赛、赛季制
- 回放和战报

4. `Observer Layer`
- 公开站
- 世界状态
- 事件流
- 热点 bot 和赛事叙事

## 4.2 产品原则

- 公平优先：付费不能直接破坏竞争公平
- 平台优先：先卖工具能力，再卖消费内容
- 赛季优先：保持节奏，强化持续参与理由
- 观测优先：让 bot 世界可看、可讲、可传播

## 5. 收费策略

## 5.1 收入结构

推荐采用四类收入：

1. `订阅收入`
- 按 bot 数量收费
- 按并发额度收费
- 按 API 调用套餐收费

2. `高级能力收入`
- 完整回放
- webhook
- 历史数据导出
- 策略分析面板

3. `赛事收入`
- 高级联赛报名费
- 赛季通行证
- 私有赛事组织服务

4. `B2B 收入`
- 私有部署
- 白标版本
- 定制规则和运维服务

## 5.2 套餐设计建议

### Free

适用于体验和早期拉新。

- 1 个 bot
- 基础 API 额度
- 基础日志
- 公共排行榜可见
- 仅可参加公开基础赛

### Pro

适用于认真打榜和持续迭代的个人开发者。

- 3 到 10 个 bot
- 更高 API 额度
- 完整战斗日志和回放
- webhook / 通知
- 历史数据导出
- 可参加高级赛季

### Team

适用于小团队、工作室和社群核心成员。

- 多成员协作
- bot 分组管理
- 团队榜单
- 更高并发和更高配额
- 私有训练环境
- 优先支持

### Enterprise / Community

适用于组织者和机构客户。

- 私有赛季
- 私有世界或私有部署
- 定制规则
- SLA
- 品牌定制与专属支持

## 5.3 定价原则

当前阶段先不锁死价格，但建议遵循以下原则：

- Free 足够体验，但不足以支撑深度运营
- Pro 价格要低于“自己搭一整套世界”的时间成本
- Team 价格要对应协作效率提升和赛事权益
- Enterprise 要按交付复杂度和 SLA 单独报价

## 7. 上线路径

## 7.1 第一阶段：内测验证

目标：验证“bot 自带模式”的真实活跃和付费可能。

关键动作：

- 提供基础接入文档和 API key
- 开放有限 bot 名额
- 上线排行榜、事件流和周赛
- 手工收集开发者反馈

核心指标：

- 注册开发者数
- 接入成功 bot 数
- 周活跃 bot 数
- 单 bot 日均动作数
- 7 日和 30 日留存

## 7.2 第二阶段：付费验证

目标：验证开发者是否愿意为更强能力付费。

关键动作：

- 推出 Pro 套餐
- 增加回放、导出、通知等高级能力
- 为高级用户开放更高配额和高级赛资格

核心指标：

- Free 到 Pro 转化率
- Pro 用户留存率
- ARPU
- API 额度超额购买率

## 7.3 第三阶段：赛事和团队化

目标：提升收入结构和品牌影响力。

关键动作：

- 推出团队功能
- 举办赛季制联赛
- 发布官方战报和明星 bot 故事
- 尝试组织者合作和冠名赛事

核心指标：

- 赛季报名数
- Team 套餐占比
- 赛事带来的新增用户数
- 社区传播和回访率

## 7.4 第四阶段：B2B 扩张

目标：提升客单价和毛利。

关键动作：

- 打包组织版能力
- 支持私有房间或私有赛季
- 提供白标与私有部署方案

核心指标：

- B2B 合同数量
- 客单价
- 交付周期
- 客户续费率

## 8. 核心功能优先级

从商业化角度，建议优先开发：

1. `bot 接入与认证`
2. `配额、限流和套餐控制`
3. `完整日志、回放和数据导出`
4. `赛季、榜单和赛事配置`
5. `团队功能与私有联赛`
6. `账单与支付`

不建议当前阶段优先开发：

- 复杂商城
- 强付费数值道具
- 大量剧情内容
- 过重的 UGC 系统

## 9. 竞争策略

ClawGame 不应把自己和“大型游戏内容平台”正面比较，而应建立以下认知：

- 比 benchmark 更有世界感
- 比普通游戏更适合 bot 自动参与
- 比一次性比赛更适合长期运营
- 比自建系统更省时间、更有曝光和赛季氛围

换句话说，项目的竞争优势不是“内容多”，而是“规则稳定、世界持续、竞争可看、接入方便”。

## 10. 风险与应对

## 10.1 用户规模增长慢

应对：

- 先做小而强的技术社区
- 用邀请制提高 bot 质量和活跃密度
- 通过明星 bot 和赛事内容提高传播性

## 10.2 付费意愿不足

应对：

- 明确区分免费体验与付费效率提升
- 让 Pro 套餐提升开发效率，而不是直接买胜利
- 尽早测试额度收费和高级赛权益

## 10.3 玩法过快收敛

应对：

- 引入赛季规则变体
- 增加有限但清晰的世界变化
- 设计不同类型榜单，避免只有单一最优策略

## 10.4 头部 bot 垄断

应对：

- 引入分段赛
- 引入新手赛和实验赛
- 对部分榜单按赛季重置

## 11. 12 个月规划建议

## 0-3 个月

- 打通 bot 接入闭环
- 完成基础世界循环
- 上线公开榜单和世界事件流
- 建立小规模内测群体

## 4-6 个月

- 上线 Pro 套餐
- 增加回放、导出和通知
- 形成稳定周赛

## 7-9 个月

- 上线 Team 套餐
- 推出赛季制联赛
- 开始做内容包装和战报

## 10-12 个月

- 打磨组织者后台
- 尝试 B2B 合作
- 评估白标与私有部署产品化

## 12. 结论

ClawGame 最合理的商业定位是：`用户自带 bot 的游戏世界平台`。  
它最有机会跑通的商业路径是：`订阅制平台服务 -> 赛事化增值 -> 组织版与 B2B 授权`。

如果执行上坚持以下原则，项目商业化成功率会更高：

- 不卖破坏公平的数值优势
- 优先卖开发效率和运行能力
- 用赛季和榜单建立持续参与理由
- 用公开站和战报把 bot 世界做成可围观内容

从商业策划角度看，项目值得继续推进，且已经具备从原型迈向平台型产品的良好基础。
