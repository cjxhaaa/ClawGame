# 战斗系统文档

## 1. 文档目标

本文件给出当前战斗系统的完整规范，作为设计、实现、测试、日志与运营观测的统一依据。

本文件只描述正式规则，不包含阶段性名称、版本标签、或待讨论项。

## 2. 战斗总览

战斗是服务器裁定的回合制系统，角色与怪物都按同一执行框架结算。

核心特征：

- 服务器权威结算
- 回合制执行
- 自动行动，不依赖手动点选
- 随机点有限且可追踪
- 全量结构化日志

标准执行链路：

1. 战斗初始化
2. 回合开始
3. 回合内行动循环
4. 回合结束
5. 胜负结算

## 3. 战斗范围与类型

系统统一支持以下战斗类型：

- `field_skirmish`：野外短战
- `elite_encounter`：精英战
- `dungeon_wave`：副本房间战斗
- `boss_phase`：Boss 阶段战斗
- `arena_duel`：竞技对抗战斗

同一战斗引擎在所有类型下复用，差异由配置控制：

- 参战模板
- 波次结构
- AI 脚本
- 失败条件
- 奖励包

## 4. 战斗实体模型

每个战斗实体必须具备以下字段：

- `entity_id`
- `team_id`
- `entity_type` (`player` / `monster` / `summon`)
- `template_id`
- `name`
- `level`
- `rank`
- `stats`
- `current_hp`
- `shield_value`
- `status_effects`
- `cooldowns`
- `equipped_skills`
- `alive`

`stats` 统一字段：

- `max_hp`
- `physical_attack`
- `magic_attack`
- `physical_defense`
- `magic_defense`
- `speed`
- `healing_power`

## 5. 战斗初始化规则

初始化阶段必须完成：

- 按模板与运行时修正生成全部实体
- 设置初始 HP（默认满值，除非遭遇配置覆盖）
- 应用入场被动与装备效果
- 锁定本场技能装配（每角色最多 4 个主动技能）
- 输出首回合可用顺序预览（可选）
- 产出战斗元信息

战斗元信息字段：

- `battle_id`
- `battle_type`
- `encounter_id`
- `region_id`
- `dungeon_id`（若存在）
- `party_size`
- `wave_index`
- `started_at`

## 6. 回合流程

每个回合固定顺序：

1. 回合开始效果
2. 计算行动顺序
3. 存活单位依次行动
4. 回合结束效果
5. 胜负检测

回合开始效果：

- 处理回合开始触发的状态与延迟效果
- 清理在回合开始到期的临时限制
- 写入回合头日志

回合结束效果：

- 持续伤害（如 `poison`、`burn`）
- 持续恢复（如 `regen`）
- 状态回合数递减
- 到期状态清理

副本战斗回合上限：

- 单场副本战斗最多 `10` 回合
- 超过回合上限仍未清场，则判定失败

## 7. 行动顺序规则

行动顺序按以下优先级决定：

1. `speed` 高者优先
2. `entity_id` 小者优先

补充规则：

- 每回合开始时重算顺序
- 速度变化影响下一回合顺序
- 跳过行动类控制不会移除单位，仅在该单位回合执行跳过

## 8. 动作体系

动作家族：

- `basic_attack`
- `skill_attack`
- `heal`
- `shield`
- `cleanse`
- `buff`
- `debuff`
- `summon`
- `wait`
- `escape`

自动行动规则：

- 玩家侧与怪物侧都按自动规则执行
- 玩家侧从“4 个已装配主动技能 + 普攻”中选取合法动作
- 非法动作（冷却中、无目标、条件不满足）不可执行

## 9. 目标选择规则

目标类型：

- `self`
- `single_ally`
- `single_enemy`
- `all_allies`
- `all_enemies`
- `front_enemy`
- `lowest_hp_ally`
- `highest_threat_enemy`
- `random_enemy`

原则：

- 玩家技能目标选择必须明确
- 怪物按脚本优先级选目标
- 随机目标必须写入日志

## 10. 仇恨与目标优先级

怪物维护对玩家侧实体的仇恨表。

仇恨生成：

- 造成伤害：`1.0x` 最终伤害
- 有效治疗：`0.45x` 有效治疗
- 施加护盾：`0.35x` 护盾值
- 嘲讽：固定仇恨注入

默认选目标链路：

1. 强制目标（脚本或嘲讽）
2. 最高仇恨存活目标
3. 执行型怪物可转向最低当前 HP 目标
4. 兜底随机存活目标

## 11. 命中与随机机制

随机机制必须受控且可回放。

规则：

- 普攻命中率 `100%`
- 技能命中率由技能定义
- 伤害浮动区间 `+/- 5%`
- 所有随机点必须落日志

允许随机点：

- 命中/未命中
- 状态施加成功/失败
- 随机目标
- 伤害浮动
- 战后掉落判定

## 12. 公式与结算顺序

物理伤害：

`final_physical = max(1, (skill_power + actor.physical_attack * atk_ratio) - target.physical_defense * def_ratio)`

魔法伤害：

`final_magic = max(1, (skill_power + actor.magic_attack * atk_ratio) - target.magic_defense * def_ratio)`

治疗：

`final_heal = max(1, skill_power + actor.healing_power * heal_ratio)`

护盾：

`final_shield = max(1, skill_power + actor.healing_power * shield_ratio)`

结算顺序：

1. 计算原始效果值
2. 应用增减益乘区
3. 护盾先吸收伤害
4. 剩余伤害扣减 HP
5. 结算附加状态

## 13. 伤害类型

伤害类型：

- `physical`
- `magic`
- `holy`
- `poison`
- `burn`
- `true`

防御关系：

- `physical` 使用 `physical_defense`
- `magic`、`holy` 使用 `magic_defense`
- `poison`、`burn` 是否穿防由技能定义
- `true` 无视双防

## 15. 冷却与动作合法性

技能级别：

- `normal`
- `advanced`
- `ultimate`

冷却规则：

- `normal`：`1` 回合
- `advanced`：`2` 回合
- `ultimate`：`3` 回合
- 普攻：`0` 回合

动作合法性：

- 冷却未结束：非法
- 无有效目标：非法
- 被 `stun`：本回合跳过
- 被 `silence`：不可用魔法标签技能
- 非本场装配技能：非法

## 16. 技能结构与自动优先级

统一技能结构字段：

- `skill_id`
- `name`
- `class`
- `weapon_style`（如有限制）
- `action_type`
- `target_type`
- `damage_type`
- `cooldown_rounds`
- `hit_chance`
- `skill_power`
- `atk_ratio`
- `def_ratio`
- `heal_ratio`
- `shield_ratio`
- `status_payloads`
- `ai_tags`

装配规则：

- 每角色战前最多装配 `4` 个主动技能
- 普攻不占技能位
- 仅允许装配与当前武器流派匹配的技能

建议自动优先级：

1. 保命技能
2. 团队恢复与护盾
3. 控制或群体技能
4. 终结技或高倍率输出
5. 普攻兜底

## 17. 怪物 AI 规则

怪物 AI 使用可解释脚本链路：

1. 强制技能指令
2. 阶段阈值分支
3. 高优先级技能条件判断
4. 默认攻击兜底

常见条件：

- 自身 HP 阈值
- 敌方人数阈值
- 阶段切换标记
- 目标状态（例如护盾、沉默）

## 18. 波次与遭遇结构

遭遇字段：

- `encounter_id`
- `battle_type`
- `region_id`
- `monster_pack`
- `waves`
- `victory_rewards`
- `failure_outcome`

波次约束：

- 野外通常 `1` 波
- 精英通常 `1-2` 波
- 副本通常普通波次 + Boss 波次
- Boss 可拆阶段但仍属同一次遭遇

## 19. 胜负结算

胜利：

- 所有敌对实体被击败
- 发放奖励
- 更新任务与成长进度
- 输出战斗摘要

失败：

- 玩家侧全灭或触发失败条件
- 执行失败后果配置

撤退：

- 是否允许撤退由战斗类型配置决定
