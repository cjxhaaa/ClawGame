# 职业技能系统

## 1. 设计目标

这份文档定义副本自动战斗所使用的职业技能系统。

核心规则：

- 游戏提供 `6` 个所有角色都可使用的通用技能
- 每个职业现在固定有 `3` 条清晰分支，用于技能分类和推荐构筑
- 每个职业仍保留少量职业内共享技能池
- 每次下副本最多装配 `4` 个主动技能
- 普通攻击始终可用，不占技能格
- 所有技能都不消耗 MP，只受冷却限制
- 副本战斗完全自动执行，并采用可解释的优先级逻辑
- 所有角色都会先经历 `civilian` 平民阶段，并在 `10` 级时解锁职业切换
- 在 V1 中，技能分支主要用于分类和推荐，不应用来写死装配
- Bot 和玩家应能更自由地混搭已解锁的通用技能与职业技能
- 非普攻技能不再单独学习，而是在冒险者公会通过升级直接解锁
- 技能支持使用金币升级，并按受控百分比提升技能效果
- 技能升级采用固定 `10` 级上限，不再随人物等级继续抬高

三职业九条路线如下：

- 战士：`tank`、`physical_burst`、`magic_burst`
- 法师：`single_burst`、`aoe_burst`、`control`
- 牧师：`healing_support`、`curse`、`summon`

## 2. 技能结构

每个技能至少需要定义：

- `skill_id`
- `name`
- `class`
- `route_id`
- `weapon_style`
- `action_type`
- `target_type`
- `damage_type`
- `skill_tier`
- `cooldown_rounds`
- `priority`
- `skill_power`
- `atk_ratio`
- `def_ratio`
- `heal_ratio`
- `shield_ratio`
- `status_payloads`
- `ai_condition`

分支规则：

- 原先硬性的 `route_id` 应弱化为分类标签，或实现层的 `skill_track`
- 分支存在的主要意义是说明玩法方向、辅助 Bot 推荐和做数值分析
- 分支不应成为技能混搭的硬限制
- 通用技能可以使用 `class = universal`、`route_id = universal`
- `weapon_style` 可以保留为兼容或风味字段，但不应继续作为限制构筑自由度的主要约束

## 3. 技能装配规则

- 每次进入副本前，角色最多装配 `4` 个主动技能
- 普通攻击固定可用，不占格子
- 同一技能不能重复装配
- 已装配技能可以从“已解锁通用技能 + 已解锁职业技能池”中自由混搭
- 角色不需要在进本前先锁定某一条路线
- Bot 应支持为不同用途预存多套自由混搭方案
- 路线只作为推荐构筑标签，不作为装配约束

冷却分层统一规则：

- `normal`：`1` 回合冷却
- `advanced`：`2` 回合冷却
- `ultimate`：`3` 回合冷却
- 普通攻击：`0` 回合冷却

## 4. 技能解锁与升级规则

### 4.1 平民阶段与解锁规则

- 普通攻击属于自带技能，角色创建时自动拥有且可直接使用
- 所有新角色初始身份都是 `civilian`
- 所有非普攻技能初始均为 `0` 级
- `0` 级技能代表未解锁，不能装配也不能使用
- 只要把技能从 `0` 级升到 `1` 级，该技能就会被解锁并变成可用
- 技能解锁与升级都在 `Adventurers Guild` 完成
- 技能解锁仍应受角色阶段、职业归属和等级要求约束

建议的解锁结构：

| 阶段 | 可解锁内容 |
| --- | --- |
| 角色创建 | 仅自带普通攻击 |
| 平民 `1-9` 级 | `6` 个通用技能可解锁到 `1` 级 |
| `10` 级后并完成转职 | 已选职业下的职业技能可继续解锁 |
| `10` 级后仍保持平民 | 继续只使用通用技能 |

设计规则：

- 只要通用技能或职业技能达到 `1` 级，就可以与当前角色允许使用的其他已解锁技能自由混搭
- 分支标签继续保留给 UI 提示、Bot 推荐和数值分析使用

### 4.2 职业选择

角色达到 `10` 级后，可以在 `Adventurers Guild` 在 `civilian`、`warrior`、`mage`、`priest` 之间切换职业；继续保持 `civilian` 也合法，后续再切换同样允许。

职业选择规则：

- 每次职业切换都会消耗 `800` 金币
- 当前职业会决定角色的职业身份
- 换职业时不会重置已学习的技能等级
- 主动技能栏会自动移除新职业下不可用的技能
- 从 `civilian` 切入正式职业时，会发放一把与职业匹配的推荐起始武器
- 路线标签只保留为职业内部技能分类，不再是第二层转职选择

职业与起始武器映射：

| 职业 | 推荐起始武器 |
| --- | --- |
| `warrior` | `sword_shield` |
| `mage` | `spellbook` |
| `priest` | `holy_tome` |

### 4.3 升级规则

- 每个技能都拥有独立的技能等级
- 升级技能需要消耗 `gold`
- 每提升 1 级，技能核心效果按受控百分比提高
- 技能等级上限固定，不随人物等级继续开放

技能上限规则：

- 所有非普攻技能初始为 `0` 级
- `0` 级代表未解锁、不可使用
- 升到 `1` 级后技能才会解锁
- 所有技能统一最高只能升到 `10` 级

建议升级效果：

- 每个技能等级提供 `+2%` 的线性成长
- “核心效果值”指伤害、治疗、护盾、召唤输出，或该技能最主要的数值效果
- 推荐公式：
  - `effective_value = base_value * (1 + 0.02 * skill_level)`
- 例子：
  - 如果某个技能基础伤害是 `攻击力 * 50%`，那么在 `1` 级时应为 `攻击力 * 50% * 1.02`
  - 同一个技能升到 `10` 级后，则应为 `攻击力 * 50% * 1.20`
- 硬控时长与硬控概率不应默认每级线性增长，除非该技能明确标注允许成长

经济规则：

- V1 中默认使用 `gold` 作为技能升级货币
- 升级消耗应随当前技能等级提高
- 本版本中，技能升级成本不应再按技能阶层区分

建议采用“升到下一级所需金币”的成本表：

| 升级目标 | 金币消耗 |
| --- | --- |
| `0 -> 1` | `120` |
| `1 -> 2` | `180` |
| `2 -> 3` | `300` |
| `3 -> 4` | `480` |
| `4 -> 5` | `750` |
| `5 -> 6` | `1140` |
| `6 -> 7` | `1680` |
| `7 -> 8` | `2400` |
| `8 -> 9` | `3360` |
| `9 -> 10` | `4620` |

平衡意图：

- 所有技能都应走同一条升级曲线，避免因为隐藏成本差异影响构筑选择
- 新技能解锁成本应低于继续深度培养已上场技能的成本
- 前几级仍应足够便宜，支持玩家和 Bot 做广泛试错
- 后几级仍然要形成有存在感的长期金币消耗
- 单技能拉满的总成本需要有存在感，但仍应低于整套终局装备词条优化的总投入

## 5. 通用技能池

通用技能对所有角色开放，也是平民阶段唯一的非普攻技能来源。

### 5.1 通用技能列表

- `Quickstep`
  - 类型：`self_buff`
  - 目标：`self`
  - 效果：获得 `speed_up` 与 `accuracy_up`，持续 `2` 回合
  - 等级：`normal`
  - 冷却：`1`

- `Pocket Sand`
  - 类型：`utility_attack`
  - 目标：`single_enemy`
  - 效果：造成轻度物理伤害，并附加 `accuracy_down`，持续 `2` 回合
  - 等级：`normal`
  - 冷却：`1`

- `Emergency Roll`
  - 类型：`defensive_mobility`
  - 目标：`self`
  - 效果：获得 `evasion_up`，并在本回合大幅降低下一次单体命中的伤害
  - 等级：`advanced`
  - 冷却：`2`

- `Signal Flare`
  - 类型：`battlefield_setup`
  - 目标：`all_enemies`
  - 效果：为所有敌人附加 `revealed`，持续 `2` 回合，使盟友对其攻击获得小幅命中加成
  - 等级：`advanced`
  - 冷却：`2`

- `Field Tonic`
  - 类型：`tempo_support`
  - 目标：`self`
  - 效果：使一个已装配技能的冷却减少 `1`，并获得短时 `speed_up`
  - 等级：`advanced`
  - 冷却：`2`

- `Tripwire Kit`
  - 类型：`trap`
  - 目标：`single_enemy`
  - 效果：放置一个延迟陷阱，在目标下一次行动时触发，造成轻度伤害并施加 `speed_down`
  - 等级：`ultimate`
  - 冷却：`3`

设计意图：

- 通用技能应当实用且有风味，但不能取代职业技能的核心身份
- 它们主要承担前期开荒、节奏处理与轻度战场准备
- 即使完成转职后，它们也应保留少量战术使用空间

## 6. 自动战斗选择逻辑

玩家侧单位每回合按以下顺序自动选动作：

1. 若自己血量低于阈值，优先保命技能
2. 若队友危险，优先治疗、护盾或召唤维持技能
3. 若敌方数量较多，优先群攻或控场技能
4. 若 Boss 进入关键阶段，优先破防、诅咒、控制或路线核心收益技能
5. 若存在高收益输出或召唤强化技能，优先使用
6. 若所有技能都在冷却，使用普通攻击

### 7.1 共享普通攻击

- `Strike`
  - 类型：`basic_attack`
  - 目标：`single_enemy`
  - 效果：基础物理攻击
  - 等级：`basic`
  - 冷却：`0`

### 7.2 Warrior 共享主动技能

- `Guard Stance`
  - 类型：`shield`
  - 目标：`self`
  - 效果：获得基于 `max_hp` 的护盾，并获得 2 回合 `defense_up`
  - 等级：`advanced`
  - 冷却：`2`

- `War Cry`
  - 类型：`buff`
  - 目标：`all_allies`
  - 效果：全队获得 2 回合 `attack_up`
  - 等级：`ultimate`
  - 冷却：`3`

- `Intercept`
  - 类型：`debuff`
  - 目标：`single_enemy`
  - 效果：造成轻度伤害并附加高仇恨嘲讽
  - 等级：`normal`
  - 冷却：`1`

### 7.3 Tank 分支

路线定位：

- 前排稳定
- 控制和减伤
- 护盾与防御叠层

路线技能：

- `Shield Bash`
  - 类型：`skill_attack`
  - 目标：`single_enemy`

  - 等级：`normal`
  - 冷却：`1`

- `Fortified Slash`
  - 类型：`skill_attack`
  - 目标：`single_enemy`
  - 效果：中等物理伤害，并给自己附加 2 回合 `defense_up`
  - 等级：`advanced`
  - 冷却：`2`

- `Bulwark Field`
  - 类型：`shield`
  - 目标：`all_allies`
  - 效果：全队获得小额护盾
  - 等级：`ultimate`
  - 冷却：`3`

- `Linebreaker`
  - 类型：`skill_attack`
  - 目标：`front_enemy`
  - 效果：对前排目标造成较高伤害，并附加 `defense_down`
  - 等级：`advanced`
  - 冷却：`2`

### 7.4 物理爆发分支

路线定位：

  - 等级：`normal`
  - 冷却：`1`

- `Blood Roar`
  - 类型：`buff`
  - 目标：`self`
  - 效果：获得 2 回合 `attack_up` 与 `speed_up`
  - 等级：`advanced`
  - 冷却：`2`

- `Execution Rush`
  - 类型：`skill_attack`
  - 目标：`single_enemy`
  - 效果：高额单体伤害，对低血目标额外增伤
  - 等级：`ultimate`
  - 冷却：`3`

## 11. 与副本节奏的关系

由于副本单场战斗最多只有 `10` 回合，所以技能设计必须满足：

- 冷却严格控制在 `1/2/3` 三档内
- 分支差异必须在前 `3-5` 回合内明显体现
- 群攻、治疗、控场、召唤存在感、保命都要能在短战斗中体现价值
- Boss 战的关键节奏应围绕 `2`、`3` 回合冷却设计

## 12. 需要后续拍板的点

- 分支标签是否要作为独立的推荐字段直接展示在 Bot 摘要和观战界面中
- 召唤物是占用明确战场槽位，还是走抽象支援实体层
- 后续是否允许同职业两个分支少量共享高阶技能
