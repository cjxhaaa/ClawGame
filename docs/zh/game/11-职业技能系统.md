# 职业技能系统

## 1. 总览

这份文档定义副本自动战斗所使用的职业技能系统。

### 1.1 核心规则

| 主题 | 规则 |
| --- | --- |
| 初始状态 | 每个新角色都会以 `civilian` 开始，拥有普攻、平民技能池，以及战士、法师、牧师三大职业的通用作战技能。 |
| 职业解锁 | 到达 `10` 级后开放转职。 |
| 装配上限 | 每次下副本最多装配 `4` 个主动技能。 |
| 普攻规则 | 普通攻击始终可用，不占主动技能格。 |
| 资源模型 | 技能不消耗 MP，只受冷却限制。 |
| 可用性规则 | 技能能否使用只由当前职业决定。 |
| 初始等级 | 所有非普攻技能初始都是 `1` 级。 |
| 升级模型 | 技能升级只强化技能效果，不承担解锁职责。 |
| 等级上限 | 所有非普攻技能统一最高 `10` 级。 |
| 构筑自由度 | 分支标签只用于分类与推荐，不构成硬性装配限制。 |

### 1.2 各职业可用技能层

| 当前职业 | 可用的非普攻技能层 |
| --- | --- |
| `civilian` | 平民技能 + 战士通用作战技能 + 法师通用作战技能 + 牧师通用作战技能 |
| `warrior` | 平民技能 + 战士通用作战技能 + 战士职业专属技能 |
| `mage` | 平民技能 + 法师通用作战技能 + 法师职业专属技能 |
| `priest` | 平民技能 + 牧师通用作战技能 + 牧师职业专属技能 |

### 1.3 职业内部路线映射

| 职业 | 路线 1 | 路线 2 | 路线 3 |
| --- | --- | --- | --- |
| `warrior` | `tank` | `physical_burst` | `magic_burst` |
| `mage` | `single_burst` | `aoe_burst` | `control` |
| `priest` | `healing_support` | `curse` | `summon` |

## 2. 技能数据结构

### 2.1 必要字段

| `weapon_style` | 可选的构筑画像标签 |

### 2.2 分类规则

| 主题 | 规则 |
| --- | --- |
| `route_id` | 用于表达玩法方向，并辅助 Bot 推荐 |
| 路线限制 | 路线标签不限制学习、升级、装配 |
| 平民层 | 平民技能使用 `class = civilian`、`route_id = civilian` |
| 职业层 | 正式职业技能使用 `class = warrior | mage | priest` |
| 武器流派 | `weapon_style` 可以保留，但不参与技能权限判断 |

## 3. 装配与可用性规则

### 3.1 装配规则

| 主题 | 规则 |
| --- | --- |
| 主动技能格 | 最多 `4` 个 |
| 普攻 | 不占格子 |
| 重复装配 | 同一技能不能重复装配 |
| 平民装配 | 平民技能 + 三大职业通用作战技能 |
| 正式职业装配 | 平民技能 + 本职业通用作战技能 + 本职业专属技能 |
| 路线混搭 | 路线标签不限制混搭 |
| Bot 方案 | Bot 应支持每个职业保存多套自由混搭方案 |

### 3.2 冷却层级

| 层级 | 冷却 |
| --- | --- |
| `basic` | `0` 回合 |
| `normal` | `1` 回合 |
| `advanced` | `2` 回合 |
| `ultimate` | `3` 回合 |

### 3.3 阶段可用范围

| 阶段 | 可用技能内容 |
| --- | --- |
| 角色创建 | 普攻 + 全部 `1` 级平民技能 + 全部 `1` 级职业通用作战技能 |
| 平民 `1-9` 级 | 持续使用平民技能与三大职业通用作战技能 |
| `10` 级后并完成转职 | 平民技能 + 所选职业通用技能 + 所选职业专属技能 |
| `10` 级后仍保持平民 | 持续使用平民技能与三大职业通用作战技能 |

## 4. 成长与升级规则

### 4.1 技能成长规则

| 主题 | 规则 |
| --- | --- |
| 初始等级 | 所有非普攻技能初始 `1` 级 |
| 记录保留 | 已学习技能等级会跨职业保留 |
| 可用性判断 | 技能只要属于当前职业可用层，就可以装配与使用 |
| 切回旧职业 | 可恢复该职业已学习技能的使用权 |
| 公会作用 | 技能升级在 `Adventurers Guild` 完成 |

### 4.2 转职规则

| 主题 | 规则 |
| --- | --- |
| 开放等级 | `10` 级 |
| 可选职业 | `civilian`、`warrior`、`mage`、`priest` |
| 金币消耗 | 每次 `800` 金币 |
| 技能等级 | 不重置 |
| 主动技能栏 | 转职时清空 |
| Bot 行为 | 转职后由 Bot 重新决定技能装配 |
| 起始武器 | 从 `civilian` 转入正式职业时发放推荐起始武器族 |

### 4.3 起始武器映射

| 职业 | 推荐起始武器 |
| --- | --- |
| `warrior` | `sword_shield` |
| `mage` | `spellbook` |
| `priest` | `holy_tome` |

### 4.4 升级规则

| 主题 | 规则 |
| --- | --- |
| 升级货币 | `gold` |
| 等级上限 | `10` |
| 数值规则 | 每升 1 级，相对 `1` 级基线增加 `+2%` 核心效果 |
| 推荐公式 | `effective_value = base_value * (1 + 0.02 * (skill_level - 1))` |
| 1 级示例 | `50% * 1.00` |
| 10 级示例 | `50% * 1.18` |
| 不建议默认线性成长 | 硬控概率与控制时长除非明确标注，否则不应自动线性成长 |

### 4.5 升级成本表

| `1 -> 2` | `180` |
| `2 -> 3` | `300` |

## 5. 平民技能层

### 5.1 平民技能层摘要

| 主题 | 规则 |
| --- | --- |
| 所属层 | `civilian` |
| 可用范围 | 所有职业共享 |
| 设计职责 | 前期开荒、节奏处理、轻度功能性支援 |
| 转职后角色 | 转职后依然属于正式职业的可混搭技能层 |

| 牧师召唤 | `Restore`, `Lantern Servitor`, `Censer Guardian`, `Choir Invocation` |

## 8. 自动战斗优先级

| 优先级 | 规则 |
| --- | --- |
| `1` | 自身血量过低时优先保命技能 |
| `2` | 队友危险时优先治疗、护盾或召唤维持技能 |
| `3` | 敌人数较多时优先群攻或战场控制技能 |
| `4` | 满足条件时优先 Boss 针对技、诅咒技或职业核心收益技能 |
| `5` | 使用当前最高收益的输出或召唤强化技能 |
| `6` | 若所有技能都在冷却，则回退到普通攻击 |

## 9. 与副本节奏的适配

| 主题 | 规则 |
| --- | --- |
| 战斗长度 | 副本战斗最多 `10` 回合 |
| 冷却结构 | 冷却应保持在 `1 / 2 / 3` 三档结构内 |
| 分支差异 | 路线差异应在第 `3-5` 回合内体现 |
| 短战斗价值 | 群攻、治疗、控场、召唤存在感与保命都要在短战斗中有价值 |
| Boss 节奏 | Boss 战的关键节奏应围绕 `2`、`3` 回合冷却设计 |
