# 战斗消耗品与药水系统

## 1. 目标

本文件定义 ClawGame 的药水驱动战斗续航模型。

它要解决当前问题：副本每过一关都自动回满血，导致战斗过于简单。

目标：

- 去掉副本房间间的自动满血恢复
- 让副本推进依赖资源规划
- 引入无声望门槛的药水商店体系
- 首批支持四类核心药水：
  - 血量恢复
  - 攻击增益
  - 防御增益
  - 速度增益
- 保持规则足够确定，便于 Bot 规划

## 2. 核心规则变更

### 2.1 关卡间不再自动回满 HP

在 V1.1 的副本房间切换中：

- 队伍成员进入下一房间时保留上一房间的当前 HP
- 进入下一房间时不再自动回满 HP
- 副本第 2 房间及之后，战斗初始化不允许隐藏免费回血

仅用于新手副本的可选缓冲规则：

- 新手副本可以配置每房间 `5% max_hp` 的过渡恢复
- 必须写入副本元数据，不能作为隐式默认行为

### 2.2 作用范围

该规则作用于：

- `dungeon_wave`
- 副本内的 `boss_phase`

除非后续单独配置，野外战斗不受此规则影响。

## 3. 药水家族

V1.1 药水家族：

1. HP 药水
   - 立即恢复生命
2. 攻击药水
   - 临时提升攻击
3. 防御药水
   - 临时提升防御
4. 速度药水
   - 临时提升速度

除 HP 药水外，其余药水都属于临时战斗增益，不提供永久成长。

## 4. 商店访问

药水购买权限不再由声望或阶位控制。

原则：

- 只要商店可访问，所有药水层级都可以购买
- 药水强度按层级递进，不依赖随机词条
- 成长压力主要来自金币成本与路线规划，而不是访问门槛

## 6. 使用规则

### 6.0 副本入场药水配置规则

进入副本前，OpenClaw 可以选择最多两种药水 ID 作为本次 run 的入场配置。

规则：

- 可以不选，也可以选择一到两种药水 ID
- 所选药水不能重复
- 本次副本自动战斗引擎只能使用这些已选药水家族
- 所选药水必须已经存在于角色背包中；进入副本不会凭空发放免费药水
- 这份药水配置会随 run 一起快照保存，后续背包变化不应回写历史 run 配置

设计目标：

- 让药水规划成为显式的副本准备步骤
- 把选择面控制在 OpenClaw 易于推理的范围内
- 避免“什么药都带上”的配置让副本准备失去权衡

### 6.1 每战斗与每次副本上限

建议上限：

- 每个角色每回合最多使用 `1` 瓶药水
- 每个角色每次副本最多使用 `3` 瓶药水
- HP 药水可在战斗内和副本内非战斗状态使用
- 增益类药水仅可在战斗中使用

### 6.2 叠加规则

为避免数值失控：

- 同家族药水效果不可叠加
- 重复使用同家族药水时，刷新持续时间并保留更高强度
- 不同家族增益可以共存
- 药水与技能是否乘区叠加，必须由状态定义显式声明

### 6.3 生效时点

- HP 药水在使用时立即结算
- 增益药水在动作结算时施加状态
- 速度药水影响下一回合排序（因为每回合重算速度顺序）

## 7. 自动战斗策略（Bot）

药水使用必须可解释、可复现。

建议默认策略：

1. 当前 HP <= `35% max_hp` 且可用时，优先使用 HP 药水
2. 预判即将承受高爆发时，使用防御药水
3. 对精英或 Boss 的关键输出窗口，使用攻击药水
4. 需要抢先手控制时，使用速度药水
5. 低风险房间不消耗非 HP 药水

所有药水决策都必须写入结构化战斗日志。

## 8. 数据模型增量

角色背包中的药水元数据应包含：

- `item_id`
- `item_type = potion`
- `potion_family`
- `tier`
- `effect_payload`
- `allowed_rank_min`
- `is_combat_only`

副本运行态应包含：

- `potions_used_by_character`
- `remaining_run_potion_uses`
- `active_potion_statuses`

战斗日志应包含：

- `potion.consumed`
- `potion.effect_applied`
- `potion.effect_expired`

## 9. 经济与商店规则

- 药水通过建筑商店购买
- 商店库存根据购买者角色阶位分层开放
- 药水价格应与修理、强化形成真实资源取舍
- 可设置每日补货，避免廉价无限续航

建议初始价格带：

- HP：低
- 防御：低-中
- 攻击：中
- 速度：中-高

## 10. 数值护栏

- 药水系统应增加策略深度，不应替代技能体系
- 单瓶药水效果不应超过同定位技能的上限
- 单次副本中的药水总价值应低于一次主要装备层级跃迁收益
- Boss 平衡应假设队伍会带部分药水，但不是满配最优药水
