## 6. 成长系统

### 6.1 职业

#### Warrior

- 定位：近战物理前排
- 优势：高生命、高防御、稳定单体输出
- 弱点：AOE 弱，自我续航有限
- 武器流派：
  - `Sword and Shield`
  - `Great Axe`

#### Mage

- 定位：爆发型魔法输出
- 优势：高 AOE、远程输出、减益能力
- 弱点：血量低、防御持续能力弱
- 武器流派：
  - `Staff`
  - `Spellbook`

#### Priest

- 定位：续航与辅助
- 优势：治疗、驱散、长线作战稳定
- 弱点：直接伤害偏低
- 武器流派：
  - `Scepter`
  - `Holy Tome`

### 6.2 声望

声望是一种可消费的委托积分，用于日常成长节奏。

规则：

- 声望主要来自每日委托提交
- 声望用于委托奖励与额外副本领奖次数购买
- 每个角色每天固定拥有 `2` 次免费副本领奖
- 每购买 `1` 次额外副本领奖需要消耗 `50` 声望
- 当天购买的额外领奖次数会在每日重置时清空，未消耗的声望余额会保留

### 6.3 角色等级

角色等级成长定义见：

- `08-赛季等级与属性框架.md`

核心规则：

- 每个赛季持续 30 天
- 赛季等级范围为 `1-100`
- 等级带来的属性成长在满级后固定
- 赛季后半段主打材料与装备追赶
- 声望作为并行经济资源，与金币和赛季成长共同构成推进节奏

## 7. 属性与战斗

这一节只保留战斗高层摘要。

完整战斗规格已经迁移到：

- `10-战斗系统框架.md`

### 7.1 核心属性

当前核心战斗属性如下：

| 字段 | 中文名 | 说明 |
| --- | --- | --- |
| `max_hp` | 最大生命值 | 角色或怪物的生命上限。 |
| `physical_attack` | 物理攻击 | 物理伤害技能与攻击的主要输出属性。 |
| `magic_attack` | 魔法攻击 | 魔法伤害技能与攻击的主要输出属性。 |
| `physical_defense` | 物理防御 | 对抗物理伤害的主要减伤属性。 |
| `magic_defense` | 魔法防御 | 对抗魔法伤害的主要减伤属性。 |
| `speed` | 速度 | 影响战斗中的行动顺序优先级。 |
| `healing_power` | 治疗强度 | 治疗技能和回复效果的主要缩放属性。 |
| `crit_rate` | 暴击率 | 影响暴击触发概率的进攻修正属性。 |
| `crit_damage` | 暴击伤害 | 暴击发生时的伤害增幅属性。 |
| `block_rate` | 格挡率 | 影响格挡结果的防御修正属性。 |
| `precision` | 精准 | 用于支撑命中稳定性与输出一致性的属性。 |
| `evasion_rate` | 闪避率 | 影响闪避结果的防御修正属性。 |
| `physical_mastery` | 物理精通 | 面向物理构筑与物理效果的补充缩放属性。 |
| `magic_mastery` | 魔法精通 | 面向魔法构筑与魔法效果的补充缩放属性。 |

相关运行时战斗字段：

| 字段 | 中文名 | 说明 |
| --- | --- | --- |
| `status_effects` | 状态效果 | 当前附着在单位身上的状态集合。 |
| `cooldowns` | 冷却状态 | 当前装配技能的剩余冷却信息。 |
| `shield_value` | 护盾值 | 先于生命值承受伤害的当前护盾量。 |

### 7.2 战斗模型

- 全部战斗采用回合制
- 战斗完全由服务器裁定
- 战斗中不包含自由移动
- 行动顺序按 `speed` 从高到低
- 若速度相同，则按更小的 `entity_id` 优先

### 7.3 命中与随机性规则

为了便于 Bot 决策：

- 普通攻击命中率固定为 `100%`
- 技能命中率由技能本身显式定义
- V1 不存在隐藏闪避属性
- 普攻不触发随机暴击
- 伤害浮动固定在 `+/- 5%`

### 7.4 伤害公式

V1 采用透明的公式家族：

- 物理伤害：
  `max(1, skill_power + actor.physical_attack * atk_ratio - target.physical_defense * def_ratio)`
- 魔法伤害：
  `max(1, skill_power + actor.magic_attack * atk_ratio - target.magic_defense * def_ratio)`
- 治疗：
  `max(1, skill_power + actor.healing_power * heal_ratio)`

战斗日志必须始终记录：

- 行动实体
- 动作名称
- 目标实体
- 原始效果类型
- 最终结算数值
- 施加或移除的状态

### 7.5 V1 状态效果

- `poison`：回合末固定伤害
- `burn`：回合末固定伤害
- `stun`：跳过下一回合
- `shielded`：优先吸收伤害
- `regen`：回合末固定治疗
- `silence`：无法使用带有魔法标签的技能

规则要求：

- 所有状态都必须明确写出持续回合数
- 同名状态默认刷新持续时间，除非特别标记为可叠加

## 8. 职业技能组

这一节只保留技能系统的高层摘要。

完整技能规格已经迁移到：

- `11-职业技能系统.md`

核心规则：

- 每个职业拥有独立技能池
- 武器流派影响属性偏向与构筑轮廓，不决定技能可用性
- 下副本时最多携带 `4` 个主动技能
- 普通攻击始终可用，且没有冷却
- 技能不消耗 MP，只受冷却回合限制
- 战斗中角色自动出手，根据技能优先级与当前战况选择动作

每种构筑的实际技能池会大于 4 个，但进副本前必须先完成技能装配。

设计目标：

- 每个职业都能通过少量动作表现明确身份
- 武器流派带来可预期的策略差异
- 不依赖隐藏组合或高频随机数才能做出有效决策
