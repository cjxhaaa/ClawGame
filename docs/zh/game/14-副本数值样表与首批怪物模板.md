# 副本数值样表与首批怪物模板

## 1. 目标

这份文档把前面的副本怪物设计继续推进成“第一批可直接实现的样表”。

当前范围：

- `Ancient Catacomb`
- `Thorned Hollow`

文档目标：

- 给出首批怪物的基础属性参考
- 给出怪物技能与冷却
- 给出 AI 使用逻辑
- 给出三挡难度下的波次排布
- 给出装备与材料掉落权重样表

说明：

- 这里的数值是第一版平衡样表，不是最终锁死值
- 重点是先把结构、比例和节奏定下来

## 2. 数值标尺

为了保持不同副本之间的可比性，先定义一个统一数值标尺。

### 2.1 T1 数值标尺

对应 `Ancient Catacomb`：

- 普通怪基础 HP：`90-150`
- 精英怪基础 HP：`220-320`
- Boss 基础 HP：`520-760`
- 普通怪输出：单次 `16-30`
- 精英怪输出：单次 `28-48`
- Boss 核心技能输出：单次 `40-72`

### 2.2 T2 数值标尺

对应 `Thorned Hollow`：

- 普通怪基础 HP：`180-260`
- 精英怪基础 HP：`360-520`
- Boss 基础 HP：`860-1180`
- 普通怪输出：单次 `28-46`
- 精英怪输出：单次 `42-68`
- Boss 核心技能输出：单次 `62-108`

## 9. 怪物 AI 模板归纳

为了后端实现，首批怪物 AI 可以先抽成 6 种模板：

| AI 模板 | 用途 | 行为特点 |
| --- | --- | --- |
| `ai_bruiser_basic` | 普通前排怪 | 优先单体输出，血低时补防御 |
| `ai_assassin_lowest_hp` | 高速刺客怪 | 优先打最低血 |
| `ai_caster_dot` | 持续伤害法系怪 | 优先挂状态，再补伤害 |
| `ai_controller_opening` | 控制怪 | 开场先上减速或易伤 |
| `ai_tank_shield_first` | 精英守卫 | 开场先护盾，再压制 |
| `ai_boss_phase_scripted` | Boss | 以阶段脚本为最高优先级 |

## 10. 后续实现建议

如果按照当前样表往后走，最合理的开发顺序是：

1. 先实现 `Ancient Catacomb easy`
2. 再补 `Ancient Catacomb hard/nightmare`
3. 然后复制框架到 `Thorned Hollow`
4. 最后再逐步扩展 `T3-T5`

这样可以最快把第一条真实副本线跑起来。
