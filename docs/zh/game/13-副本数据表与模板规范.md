# 副本数据表与模板规范

## 1. 目标

本文件定义“多怪编队副本系统”的数据结构，确保以下能力可配置：

- 每房多怪组合
- 普通怪 / 精英怪 / Boss 分层
- easy / hard / nightmare 三档难度
- Boss 仅在最后房间出现
- 副本套装身份与 Bot 可读的产出摘要
- 四个并行赛季副本共享统一奖励预算

## 2. 数据分层

建议拆为七层：

1. 副本定义层
2. 难度配置层
3. 房间强度层
4. 怪物模板层
5. 房间编队层
6. Boss 阶段脚本层
7. 奖励掉落层

## 3. 核心表清单

- `dungeon_definitions`
- `dungeon_difficulty_profiles`
- `dungeon_room_configs`
- `dungeon_monster_templates`
- `dungeon_monster_skills`
- `dungeon_room_waves`
- `dungeon_wave_slots`
- `dungeon_boss_phases`
- `dungeon_boss_phase_actions`
- `dungeon_rating_reward_tables`
- `monster_material_drop_tables`

## 4. dungeon_definitions

用于定义副本本体。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | text | 副本 ID |
| `name` | text | 名称 |
| `region_id` | text | 区域 ID |
| `room_count` | int | 固定 `6` |
| `boss_room_index` | int | 固定 `6` |
| `min_rank` | text | 最低阶位 |
| `set_id` | text | 副本套装归属 |
| `set_identity_tags` | text[] | 面向 Bot 的产出标签，例如 defense、crit、spell |
| `weapon_pool_summary` | jsonb | 可见武器池摘要 |
| `armor_pool_summary` | jsonb | 可见防具池摘要 |
| `material_identity` | text | 材料主题摘要 |
| `recommended_power_floor` | int | 推荐准备战力下限 |
| `recommended_power_mid` | int | 推荐稳定通关战力 |
| `recommended_power_ceiling` | int | 推荐碾压战力 |

## 5. dungeon_difficulty_profiles

定义副本三档难度的全局系数。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | text | 难度配置 ID |
| `dungeon_id` | text | 所属副本 |
| `difficulty` | text | `easy` / `hard` / `nightmare` |
| `hp_multiplier` | numeric | 全局生命倍率 |
| `damage_multiplier` | numeric | 全局伤害倍率 |
| `defense_multiplier` | numeric | 全局防御倍率 |
| `speed_multiplier` | numeric | 全局速度倍率 |
| `elite_spawn_bias` | numeric | 精英出现概率偏置 |
| `mechanic_intensity` | text | `low` / `mid` / `high` |

## 6. dungeon_room_configs

定义每个房间在每个难度下的压力目标。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | text | 房间配置 ID |
| `dungeon_id` | text | 所属副本 |
| `difficulty` | text | 难度 |
| `room_index` | int | 房间序号 `1-6` |
| `room_type` | text | `normal` / `elite` / `boss` |
| `target_monster_count_min` | int | 最少怪物数 |
| `target_monster_count_max` | int | 最多怪物数 |
| `must_have_elite` | bool | 是否必须有精英怪 |
| `allow_boss` | bool | 仅 `room_index=6` 时为 true |
| `rating_if_failed_here` | text | 失败评级 |

约束：

- `allow_boss = true` 仅允许 `room_index = 6`

| `name` | text | 名称 |
| `monster_tier` | text | `normal` / `elite` / `boss` |

## 8. dungeon_room_waves

定义每个房间的波次。V1 可先做单波，后续扩展多波。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | text | 波次 ID |
| `dungeon_id` | text | 所属副本 |
| `difficulty` | text | 难度 |
| `room_index` | int | 房间序号 |
| `wave_index` | int | 波次序号 |
| `is_boss_wave` | bool | 是否 Boss 波 |

## 9. dungeon_wave_slots

定义波次内具体怪物槽位。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | text | 槽位 ID |
| `wave_id` | text | 所属波次 |
| `slot_index` | int | 槽位序号 |
| `monster_id` | text | 怪物 ID |
| `count` | int | 数量 |
| `spawn_timing` | text | `start` / `round_2` / `phase_2` |

约束：

- `room_index < 6` 时，`monster_tier = boss` 禁止出现
- `room_index = 6` 时，必须至少有 1 个 `monster_tier = boss`

## 12. 奖励与掉落

- 装备奖励继续按评级表结算
- 材料掉落继续按怪物击杀表结算
- `nightmare` 可提高高阶材料权重，但不改变“Boss 只在末房”的结构规则
- 四个赛季副本应共享同一套品质分布与奖励形状逻辑
- 选择副本时，主要变化的是 `set_id`、产出标签和材料主题
- 职业限制武器仍属于共享掉落池，不按调用者职业过滤

## 13. 样例约束（必须在程序校验）

1. 每个副本必须存在 `easy/hard/nightmare` 三条 difficulty profile
2. 每个难度必须完整配置 `room_index=1..6`
3. `room_index=6` 必须包含 Boss
4. `room_index<6` 不得包含 Boss
5. `nightmare` 的房间怪物总数、精英密度或机制密度必须高于 `hard`
6. 每个副本必须暴露 `set_id` 与 `set_identity_tags`
7. 每个副本必须暴露包含六种武器流派的共享武器池摘要
