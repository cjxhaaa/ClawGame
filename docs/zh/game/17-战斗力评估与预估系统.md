# 战斗力评估与预估系统

## 1. 目标

本规范定义一个统一、可解释、可对比的战斗力数值系统，用于三类场景：

- 角色面板展示：让玩家快速理解当前强度
- 装备价值展示：每件装备有独立战斗力贡献
- 对战前预估：快速判断副本通关把握与竞技场胜率区间

该系统是评估系统，不替代真实战斗推演结果。

## 2. 设计原则

- 可解释：每个分数都有来源
- 可拆解：总分可拆为角色基础分、装备分、技能修正分
- 可比较：同阶段角色分数具备横向可比性
- 可扩展：后续新增词条、套装、天赋时可增量接入
- 可控波动：避免换一件小装备导致总分剧烈跳动

## 3. 总评分模型

角色总战斗力定义为：

总战斗力 = 基础成长分 + 装备总分 + 构筑修正分 + 场景修正分

其中：

- 基础成长分：等级、阶位、职业基础属性形成的稳定底盘
- 装备总分：所有已装备道具评分之和
- 构筑修正分：技能搭配、药水准备、抗性覆盖等战斗准备质量
- 场景修正分：按副本或竞技场场景做小幅修正，用于预估

默认面板展示值使用：

- 面板战斗力 = 基础成长分 + 装备总分 + 构筑修正分

不把场景修正分写入通用面板，避免 UI 频繁变化。

## 4. 基础成长分

## 4.1 阶位系数

- low: 1.00
- mid: 1.35
- high: 1.75

## 4.2 等级成长

基础等级分：

等级分 = 20 × 等级

平滑增长加成：

平滑分 = 8 × sqrt(等级)

## 4.3 属性折算

属性分使用角色面板主属性折算：

属性分 =
1.2 × max_hp
+ 6.0 × 主攻击
+ 4.5 × 次攻击
+ 5.0 × physical_defense
+ 5.0 × magic_defense
+ 4.0 × speed
+ 4.0 × healing_power

职业主攻击定义：

- warrior: physical_attack
- mage: magic_attack
- priest: magic_attack

基础成长分最终公式：

基础成长分 = 阶位系数 × (等级分 + 平滑分 + 属性分)

## 5. 单件装备评分

每件装备评分定义为：

装备评分 = 品质基值 + 主词条分 + 副词条分 + 强化分 + 套装分摊

## 5.1 品质基值

- common: 30
- uncommon: 55
- rare: 90
- epic: 145
- legendary: 220
- mythic: 320

## 5.2 主词条与副词条折算系数

主词条与角色属性同系数：

- max_hp: 1.2
- physical_attack: 6.0
- magic_attack: 6.0
- physical_defense: 5.0
- magic_defense: 5.0
- speed: 4.0
- healing_power: 4.0

副词条权重：

- 有效副词条：1.0 倍
- 半有效副词条：0.55 倍
- 无效副词条：0.25 倍

职业有效性判定示例：

- warrior 的 magic_attack 属于半有效
- mage 的 physical_attack 属于半有效
- priest 的双防通常有效

## 5.3 强化分

强化分采用递增但受控曲线：

强化分 = 12 × 强化等级 + 1.8 × 强化等级^2

## 5.4 套装分摊

成套效果总加分按件数均摊到每件：

- 2 件套激活：总计 +60
- 4 件套激活：额外 +140
- 6 件套激活：额外 +220

## 6. 装备总分

装备总分 = 所有已装备部位装备评分求和

建议显示两层：

- 当前装备总分
- 可替换候选装备的评分对比（例如 +37 或 -12）

## 7. 构筑修正分

构筑修正分范围建议控制在总分的 -8% 到 +12%。

## 7.1 技能组修正

- 技能槽位覆盖完整且冷却结构合理：+0 到 +120
- 输出与生存严重失衡：-0 到 -80

## 7.2 药水准备修正

基于副本或竞技场前可用药水：

- 满足推荐药水组合：+40 到 +140
- 缺乏关键药水：0 到 -90

## 7.3 抗性与短板修正

对关键短板进行扣分：

- speed 低于场景阈值较多：最多 -120
- 双防明显偏低：最多 -160

## 8. 场景预估模型

## 8.1 副本预估战力

对目标副本定义推荐战力区间：

- 推荐下限: P50 通关线
- 推荐中位: P70 稳定通关线
- 推荐上限: P90 高速通关线

计算场景战力：

副本战力 = 面板战斗力 + 副本场景修正

通关概率分层建议：

- < 下限: 低（<35%）
- 下限到中位: 中（35%-65%）
- 中位到上限: 高（65%-85%）
- > 上限: 很高（>85%）

## 8.2 竞技场预估强度

设双方场景战力分别为 A 与 B，定义战力差：

Delta = A - B

将 Delta 映射到胜率：

胜率(A) = 1 / (1 + exp(-Delta / 420))

为了避免 UI 过度确定，前端展示区间而非点值：

- 极劣势: <30%
- 劣势: 30%-45%
- 接近: 45%-55%
- 优势: 55%-70%
- 极优势: >70%

## 9. UI 展示规范

## 9.1 角色状态栏优先展示

角色状态栏首屏应优先显示：

- 战斗力总分（大数字）
- 相对上次变动值（例如 +126）
- 当前场景预估标签（副本: 中等把握 / 竞技场: 小优）

## 9.2 角色面板分解

角色面板展示三段分解：

- 基础成长分
- 装备总分
- 构筑修正分

支持展开明细查看每项来源。

## 9.3 装备卡片展示

每件装备展示：

- 装备战斗力评分
- 相比当前同部位装备差值
- 该装备对总分影响预估

## 10. 数值标定建议

建议建立三条监控曲线：

- 同等级不同职业平均战斗力曲线
- 同阶段副本推荐战力与真实通关率对应曲线
- 竞技场战力差与真实胜率拟合曲线

每个赛季最少做一次回归标定，防止评分系统与真实强度漂移。

## 11. 示例

示例角色：

- 等级 30，rank = mid
- 基础成长分 = 4280
- 装备总分 = 2360
- 构筑修正分 = 180

则面板战斗力：

- 6820

进入某副本前，场景修正为 -120：

- 副本战力 = 6700
- 该副本推荐区间为 6400 到 7200
- 预估为中高把握

## 12. 与现有系统的接入点

- 角色基础属性来自角色状态快照
- 装备评分在装备服务中逐件计算并缓存
- 副本推荐阈值来自副本定义表
- 竞技场预估在匹配或预览时按双方实时面板计算

## 13. 非目标

本系统不做以下事项：

- 不替代真实战斗推演与日志
- 不直接决定奖励，只做预估与展示
- 不作为反作弊唯一依据

## 14. 版本管理

- 评分公式版本号建议独立维护，例如 power_score_v1_0
- 后端响应中返回公式版本，便于前后端与回放解释一致
