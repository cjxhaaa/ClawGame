# 游戏规格拆分索引

这个目录是 `game-spec-v1.md` 的模块化拆分版本。

文件说明：

- `01-概览.md`
  产品定位、范围、核心循环、时间规则
- `02-成长与战斗.md`
  职业、阶位、属性、战斗、技能组
- `03-世界系统.md`
  装备、经济、地图、建筑、任务、地下城、竞技场
- `04-bot与平台.md`
  Bot 集成、事件模型、后端架构、数据模型、API 质量要求
- `05-网站运维与交付.md`
  官网规格、观测、安全、测试、验收标准、交付阶段
- `06-世界地图定义.md`
  单独的世界地图定义，补充地理结构、旅行线路、区域身份与官网地图呈现规则
- `07-地点目录与产出定义.md`
  面向观测站官网的地点目录，定义地点命名、背景、NPC、设施、地牢关系与材料产出
- `08-赛季等级与属性框架.md`
  定义赛季周期、等级经验曲线、等级成长属性，以及后续装备系统所需的数值底座
- `09-装备、副本与掉落体系.md`
  定义装备槽位、共享赛季掉落池、四个并行副本套装方向、跨职业武器随机掉落，以及按 `S/A/B/C/D/E` 评级发放的副本装备奖励
- `10-战斗系统框架.md`
  定义回合流程、动作结算、目标选择、状态规则、AI 行为约束，以及后续副本怪物所需的战斗日志契约
- `11-职业技能系统.md`
  定义三大职业与两种武器流派的技能池、四技能携带规则，以及自动战斗时的技能选择逻辑
- `12-副本怪物与难度系统.md`
  定义副本怪物模板、最多 `6` 个递进房间、Boss 阶段、评级规则，以及“装备按评级结算、材料按击杀掉落”的产出逻辑
- `13-副本数据表与模板规范.md`
  把副本、房间、怪物、Boss 脚本、评级装备奖励和怪物材料掉落进一步整理成可落地的数据表结构
- `14-副本数值样表与首批怪物模板.md`
  提供 Ancient Catacomb 与 Thorned Hollow 的首批怪物属性、技能、AI、波次与掉落权重样表
- `15-战斗消耗品与药水系统.md`
  去掉副本房间间自动满血，并定义按阶位解锁的 HP/攻击/防御/速度药水体系
- `16-自动战斗推演流程说明.md`
  逐步说明自动战斗从初始化到回合循环、动作结算、日志输出、胜负结算的完整流程
- `17-战斗力评估与预估系统.md`
  定义角色面板战斗力、单件装备评分、总分计算，以及副本与竞技场的强度预估模型

说明：

- 完整合并版仍保留在 `docs/zh/game-spec-v1.md`。
