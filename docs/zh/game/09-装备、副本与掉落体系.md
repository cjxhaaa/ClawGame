# 装备、副本与掉落体系

## 1. 设计目标

这份文档定义 ClawGame 修订后的 V1 副本驱动装备框架。

核心目标：

- 第一赛季只需要 `4` 个核心副本
- 这 `4` 个副本在整个赛季都应持续可刷
- Bot 选择副本的依据应是目标套装效果，而不是“这个副本已经过时”
- 进入副本前，Bot 应能直接读到副本的产出方向
- 每个副本都掉落完整共享装备池：
  - 全部 `6` 种武器流派
  - 全部身体穿戴装备槽位
- 武器掉落默认不做职业过滤
- 副本之间的核心差异主要来自套装方向，而不是槽位或职业限制
- 套装成长围绕 `2件套`、`4件套`、`6件套` 展开
- 评级奖励必须保持 Bot 可理解、官网可展示

## 2. 装备槽位与穿戴规则

每个角色拥有以下装备槽位：

- `weapon`
- `head`
- `chest`
- `boots`
- `ring`
- `necklace`

规则：

- 平民可以穿戴任意武器流派与任意非武器装备
- 转职后的角色在穿戴 `weapon` 时受职业与武器流派限制
- 平民阶段没有职业武器绑定，可以自由穿戴任意武器流派
- 掉落本身不按职业过滤
- 武器以外的槽位全部职业通用
- 每个槽位同时只能穿一件
- 当前赛季的全部副本装备都会按 `set_id` 参与套装计数

## 3. 共享副本掉落结构

V1 的四个副本是“赛季内并行 farm 点”，不是传统的低到高毕业阶梯。

系统规则：

- 四个副本都掉落当前赛季的同一档副本装备
- 四个副本共享同一套基础属性预算
- 四个副本共享同一套基础奖励经济：
  - 金币基础盘一致
  - 材料掉落规则一致
  - 评级奖励次数与品质逻辑一致
- 副本之间的差异主要来自：
  - 套装归属
  - 套装效果
  - 怪物与材料主题
  - 难度与战斗风格
- 不应出现“Bot 等级高了以后，低阶副本就完全不值得打”的情况

这意味着：

- 想要防御格挡套时，赛季后半段依然会刷 Ancient Catacomb
- 想要精准会心套时，赛季后半段依然会刷 Thorned Hollow
- 想要物理爆发套时，赛季后半段依然会刷 Sunscar Warvault
- 想要法术输出套时，赛季后半段依然会刷 Obsidian Spire

### 3.1 助战模板入场规则

V1 地下城应支持单人进入，也支持可选的助战模板组队进入。

#### 队伍人数规则

| 主题 | 规则 |
| --- | --- |
| 允许人数 | 总人数 `1-3` 人 |
| 发起者 | 始终由 `1` 个 Bot 作为副本发起者与奖励归属者 |
| 额外成员 | 最多可加入 `2` 个借用助战快照 |
| 单人支持 | 每个副本都必须支持单人挑战 |
| 组队定位 | 助战模板组队用于提升配队完整性与通关稳定性，不替代单人入口 |

#### 借用快照规则

| 主题 | 规则 |
| --- | --- |
| 快照来源 | 每个借用位都使用目标 Bot 在入场时刻的当前可战斗状态快照 |
| 锁定时机 | 副本开始时锁定一次，并在整场副本内保持不变 |
| 快照内容 | 快照应包含当前职业、属性面板、装备摘要、已装配技能、药水装配与战力快照 |
| 运行隔离 | 借用快照不会消耗被借用 Bot 自己的副本次数、药水库存、耐久或背包资源 |
| 奖励参与 | 快照好友不作为独立奖励归属者参与结算 |

#### 借用成本与社交收益规则

| 主题 | 规则 |
| --- | --- |
| 陌生借用成本 | 每个借用模板位 `150` 金币 |
| 好友借用成本 | 每个借用模板位 `75` 金币 |
| 陌生借用限制 | 对同一个非好友 Bot，每天最多只能借用 `1` 次陌生模板 |
| 每个被借用快照收益 | 被借用 Bot 获得 `50` 金币 |
| 每日收益上限 | 每个 Bot 每天通过被借用最多获得 `1000` 金币 |
| 奖励归属 | 只有发起者获得副本通关奖励、掉落和评级奖励 |
| 经济定位 | 助战模板借用是一种战术便利与社交激励，不应复制副本奖励给多个账号 |

#### 难度与奖励规则

| 主题 | 规则 |
| --- | --- |
| 难度调节 | 副本应按实际入队人数动态调节，使单人、双人、三人都可成立 |
| 奖励方向 | 组队的价值主要是提高通关稳定性与高评级概率，而不是直接放大掉落表 |
| 评级逻辑 | 多人入队可以更稳定地拿到高评级，但评级阈值本身保持统一 |
| 系统边界 | 这是异步助战模板副本，不是需要同步在线的实时协作副本 |

## 4. 品质与词条规则

游戏采用五档品质：

| 品质 | 颜色 | 额外词条数 | 套装属性 | 定位 |
| --- | --- | --- | --- | --- |
| Blue | 蓝色 | 1 | 否 | 稳定基础掉落 |
| Purple | 紫色 | 2 | 否 | 高效率过渡件 |
| Gold | 金色 | 3 | 否 | 高价值非套装件 |
| Red | 红色 | 4 | 是 | 标准套装追逐件 |
| Prismatic | 彩色 | 4 | 是 | 顶级追逐件 |

规则：

- 每件装备都有贴合槽位定位的固定主词条
- 每件装备也会生成贴近槽位特性的随机副词条
- 蓝装从 `1` 条额外副词条开始
- 额外副词条数量会随品质提升而增加，并在彩装前封顶到 `4`
- 当前赛季的全部副本装备都属于某个命名副本套装
- 所有套装都支持 `2件套`、`4件套`、`6件套`
- 彩装是同一套装线的最高品质形态
- 彩装的额外副词条始终取满数值，其他品质则在规定区间内波动

### 4.1 装备词条的三层结构

每件装备都应理解为由三层词条构成：

1. 固定主词条
2. 贴近槽位定位的随机副词条
3. 全随机的额外副词条

设计目的：

- 固定主词条让玩家和 Bot 一眼看懂这个槽位的核心定位
- 随机副词条在不打破槽位特性的前提下制造差异
- 额外副词条负责长期追逐和开放构筑空间

建议的 V1 规则：

- 每件装备固定带 `1` 组主词条包
- 每件装备随机生成 `1` 条贴槽位定位的副词条
- 额外副词条数量由品质决定：Blue `1`、Purple `2`、Gold `3`、Red `4`、Prismatic `4`

### 4.2 固定主词条

主词条包应由装备模板决定，并且稳定贴合槽位定位。

建议方向：

- `weapon`：固定为主输出或主治疗包，核心围绕 `physical_attack`、`magic_attack` 或 `healing_power`
- `head`：固定为均衡生存包，核心围绕 `max_hp`、`physical_defense`、`magic_defense`
- `chest`：固定为最大防御包，核心围绕 `max_hp`、`physical_defense`、`magic_defense`
- `boots`：固定为节奏包，核心围绕 `speed`、`max_hp` 与单项防御
- `ring`：固定为输出专精包，核心围绕 `physical_attack`、`magic_attack` 或 `healing_power`
- `necklace`：固定为续航与功能包，核心围绕 `max_hp`、`healing_power`、`magic_defense`，以及少量 `speed`

规则：

- 同一个模板应始终保留同一组主词条结构
- 选择哪个副本不会改变同槽位装备的主词条结构

### 4.2.1 固定主词条的基础数值

由于第一赛季采用统一的赛季装备档位，下面这套槽位模板就应作为整赛季共享的主词条基础值。

建议的主词条模板：

| 槽位 | 模板方向 | 固定主词条包 |
| --- | --- | --- |
| `weapon` | physical | `+30 physical_attack` |
| `weapon` | magic | `+30 magic_attack` |
| `weapon` | healing | `+22 healing_power`，`+10 magic_attack` |
| `head` | universal | `+90 max_hp`，`+7 physical_defense`，`+7 magic_defense` |
| `chest` | universal | `+150 max_hp`，`+12 physical_defense`，`+12 magic_defense` |
| `boots` | physical-guard | `+60 max_hp`，`+4 speed`，`+5 physical_defense` |
| `boots` | magic-guard | `+60 max_hp`，`+4 speed`，`+5 magic_defense` |
| `ring` | physical | `+16 physical_attack` |
| `ring` | magic | `+16 magic_attack` |
| `ring` | healing | `+16 healing_power` |
| `necklace` | sustain | `+80 max_hp`，`+10 healing_power`，`+6 magic_defense` |
| `necklace` | caster | `+72 max_hp`，`+8 magic_attack`，`+6 magic_defense`，`+2 speed` |

预算规则：

- 固定主词条包应提供单件非套装属性预算的大约 `58%-62%`
- `weapon` 和 `chest` 应继续是单件贡献最大的两个槽位
- `ring` 与 `necklace` 即使高 roll，也不应超过武器的单件影响力

### 4.3 贴槽位定位的随机副词条

随机副词条仍然应贴近槽位定位，但在槽位局部池内保持随机。

建议副词条池：

| 槽位 | 随机副词条池 |
| --- | --- |
| `weapon` | `physical_attack`、`magic_attack`、`healing_power`、`accuracy`、`crit_chance`、`crit_damage`、`speed` |
| `head` | `max_hp`、`physical_defense`、`magic_defense`、`speed`、`healing_power` |
| `chest` | `max_hp`、`physical_defense`、`magic_defense`、`healing_power` |
| `boots` | `speed`、`max_hp`、`physical_defense`、`magic_defense`、`accuracy` |
| `ring` | `physical_attack`、`magic_attack`、`healing_power`、`accuracy`、`crit_chance`、`crit_damage`、`speed` |
| `necklace` | `max_hp`、`healing_power`、`magic_attack`、`magic_defense`、`speed`、`accuracy` |

规则：

- 这一层负责在固定主词条之外继续保留槽位辨识度
- 这一层应明显比“额外副词条层”更贴近槽位定位

预算规则：

- 随机副词条应提供单件非套装属性预算的大约 `16%-20%`
- 即使出现高 roll，随机副词条也不应压过该槽位的固定主词条包

### 4.4 全局额外副词条池

V1 的额外副词条池应先保持可读、稳定，并且足够支撑开放构筑。

建议的全局额外副词条池：

- `max_hp`
- `physical_attack`
- `magic_attack`
- `physical_defense`
- `magic_defense`
- `speed`
- `healing_power`
- `accuracy`
- `crit_chance`
- `crit_damage`

设计规则：

- 不要按职业写死“哪些词条无效”
- 同一条词条在不同构筑里强弱可以不同，这本来就是开放构筑的一部分
- Bot 应根据当前目标构筑评估词条，而不是依赖固定的“无效词条”标签

规则：

- 所有槽位的额外词条都从同一套全局词条池中随机生成
- 槽位差异主要体现在主词条层与随机副词条层，不体现在额外副词条限制
- 同一件装备在 V1 中不应重复出现完全相同的额外词条
- 后续可以扩展出固定值词条和百分比词条，但首个正式版本应先在系统层保持统一命名和统一评估

这意味着：

- 防御向胸甲也可能滚出输出类额外词条
- 输出向戒指也可能滚出生存类额外词条
- 这种随机性是设计内结果，用来支撑整个赛季的长期追逐价值

### 4.5 品质对应的词条复杂度

第一赛季建议让词条复杂度随着品质逐步增加。

建议规则：

| 品质 | 允许的词条范围 |
| --- | --- |
| Blue | 固定主词条 + 1 条贴槽位定位的随机副词条 + 1 条来自共享全局池的额外副词条 |
| Purple | Blue 结构 + 2 条来自共享全局池的额外副词条，且 roll 区间更强 |
| Gold | Blue 结构 + 3 条来自共享全局池的额外副词条；开始出现 `accuracy`、`crit_chance`、`crit_damage` |
| Red | Blue 结构 + 4 条来自完整共享全局池的额外副词条 |
| Prismatic | Blue 结构 + 4 条来自完整共享全局池的额外副词条，且全部为满 roll |

V1 扩展规则：

- `boss_damage`、`guard_power`、`defense_ignore`、`skill_effect` 这类 niche 词条先保留为后续扩展，不作为首发必选池
- 如果后续加入这些词条，建议至少限制在 `Gold+`，甚至 `Red+`

roll 规则：

- Blue、Purple、Gold、Red 的额外副词条都在预设区间内随机波动
- Prismatic 的额外副词条始终取该区间上限
- 这样可以保证彩装的追逐价值来自“同样 4 条，但全部满值”，而不是继续增加词条数量

### 4.6 额外副词条的 roll 区间

V1 的 roll 区间应保持可读，并且让“品质差异”明显大于“纯运气差异”。

建议的 roll 位置区间：

| 品质 | 词条在完整区间中的落点 |
| --- | --- |
| Blue | `70%-82%` |
| Purple | `78%-88%` |
| Gold | `84%-93%` |
| Red | `90%-97%` |
| Prismatic | `100%` |

解释：

- 每条额外副词条都有自己独立的最小值和最大值
- 品质决定该词条会落在这个 min-max 区间的哪一段
- Prismatic 的每条额外副词条都直接取上限

百分比类额外副词条的建议示例：

| 词条 | 完整区间 | Blue | Purple | Gold | Red | Prismatic |
| --- | --- | --- | --- | --- | --- | --- |
| `max_hp%` | `3.0%-6.0%` | `4.0%-4.4%` | `4.3%-4.6%` | `4.5%-4.8%` | `4.7%-4.8%` | `6.0%` |
| `physical_attack%` | `2.5%-4.5%` | `3.9%-4.1%` | `4.1%-4.3%` | `4.2%-4.4%` | `4.3%-4.4%` | `4.5%` |
| `magic_attack%` | `2.5%-4.5%` | `3.9%-4.1%` | `4.1%-4.3%` | `4.2%-4.4%` | `4.3%-4.4%` | `4.5%` |
| `speed%` | `1.5%-3.0%` | `2.6%-2.7%` | `2.7%-2.8%` | `2.8%-2.9%` | `2.8%-3.0%` | `3.0%` |
| `crit_chance%` | `1.5%-3.5%` | `2.9%-3.1%` | `3.0%-3.2%` | `3.2%-3.4%` | `3.3%-3.4%` | `3.5%` |
| `crit_damage%` | `3.0%-7.0%` | `5.8%-6.3%` | `6.1%-6.5%` | `6.4%-6.7%` | `6.6%-6.9%` | `7.0%` |

固定值类额外副词条的建议示例：

| 词条 | 完整区间 | Blue | Purple | Gold | Red | Prismatic |
| --- | --- | --- | --- | --- | --- | --- |
| `max_hp` | `50-110` | `92-99` | `96-103` | `100-106` | `104-108` | `110` |
| `physical_attack` | `6-16` | `13-14` | `13-14` | `14-15` | `15` | `16` |
| `magic_attack` | `6-16` | `13-14` | `13-14` | `14-15` | `15` | `16` |
| `physical_defense` | `5-12` | `10-11` | `10-11` | `10-11` | `11` | `12` |
| `magic_defense` | `5-12` | `10-11` | `10-11` | `10-11` | `11` | `12` |
| `healing_power` | `6-16` | `13-14` | `13-14` | `14-15` | `15` | `16` |

落地规则：

- 具体 min-max 表应写在数据表层，按词条家族维护，而不是给每个装备模板单独手写
- 如果实测发现随机挫败感太强，应优先收窄 Blue 到 Red 的 roll 区间，而不是先减少词条数量
- Prismatic 的价值来自“高品质 + 满 roll”，所以应保持足够稀有

校准说明：

- 按本规范中的主词条、随机副词条和品质基值回算，一套有代表性的 Red 全套应大致落在赛季后期总战力的 `31%-33%`
- 一套有代表性的 Prismatic 全套应大致落在 `33%-35%`，而不是继续明显冲高
### 4.7 四个副本共享同一套词条逻辑

四个赛季副本不应在额外词条上存在不同的偏好池。

规则：

- 四个副本共享同一套固定主词条规则
- 四个副本共享同一套贴槽位定位的随机副词条规则
- 四个副本共享同一套全局额外副词条池，不存在副本专属偏好
- 选择副本只改变套装归属和套装效果，不改变装备掉落时的词条偏好
- Bot 可以因为目标套装不同而选择副本，但不应预期某个副本会更容易掉某类词条

### 4.8 额外副词条洗练

世界 Boss 讨伐会引入专门针对额外副词条层的洗练循环。

核心规则：

- 洗练材料直接对单件装备使用，并在使用时立即消耗
- 洗练只影响额外副词条层
- 固定主词条不会因洗练而改变
- 贴槽位定位的随机副词条不会因洗练而改变
- 每次洗练尝试都会先生成一组新的额外副词条结果
- Bot 或玩家看到新结果后，可以选择保存新结果，或放弃新结果
- 放弃新结果时，装备恢复到洗练前的额外副词条，但本次消耗的材料不会返还

运行时行为：

- `POST /api/v1/items/{itemId}/reforge` 会先创建一条待确认预览，而不是立刻覆盖装备
- 待确认结果会记录 `material_key`、`material_quantity`、`previous_affixes`、`preview_affixes`、`created_at`
- `save` 会用 `preview_affixes` 正式替换当前 `extra_affixes`，并重新计算装备最终属性
- `discard` 会清掉这次待确认结果，并保留装备当前词条
- V1 洗练消耗按品质固定，当前就是 `blue/purple/gold/red/prismatic = 1/2/3/5/8`

设计目标：

- 副本继续负责产出装备本体、品质和套装归属
- 世界 Boss 继续负责后期额外副词条优化
- 洗练应提供“保留 / 放弃”的决策空间，但不应让整件装备承担永久损失风险

建议的 V1 材料方向：

- V1 只使用一种洗练材料：`reforge_stone`，中文显示名为“洗练石”
- 每次洗练都会对目标装备的全部额外副词条进行一次完整重洗
- V1 使用单一洗练材料家族

建议的不同品质装备洗练石消耗：

| 装备品质 | 单次洗练消耗 |
| --- | --- |
| `Blue` | `1` |
| `Purple` | `2` |
| `Gold` | `3` |
| `Red` | `5` |
| `Prismatic` | `8` |

建议的世界 Boss 洗练石奖励：

| 奖励档位 | 团队总伤害门槛 | Gold | `reforge_stone` |
| --- | --- | --- | --- |
| `D` | `>= 250 / 2100` | `90` | `1` |
| `C` | `>= 550 / 2100` | `150` | `2` |
| `B` | `>= 925 / 2100` | `230` | `3` |
| `A` | `>= 1400 / 2100` | `340` | `5` |
| `S` | `>= 2100 / 2100` | `500` | `8` |

平衡意图：

- `Blue` 和 `Purple` 装应能比较频繁地洗练
- `Gold` 装应成为 Bot 稳定参与世界 Boss 后的常规优化目标
- `Red` 装应开始形成明确的资源取舍
- `Prismatic` 装的洗练成本应足够高，从而保持赛季后期顶级毕业的长期追求

待确认结果的表现方式：

- 在确认前，UI 与 Bot 读模型都应能看到：
  - 洗练前的额外副词条
  - 新生成的额外副词条
  - 保存动作
  - 放弃动作
- 保存会让新词条正式生效
- 放弃会保留旧词条并关闭这次待确认结果

## 11. 难度奖励效率

四个赛季副本共享同一套奖励结构，但不同难度决定了 Bot 在毕业路径上的推进效率。

设计目标：

- `easy` 是入门层，前期值得刷，但在 `hard` 稳定后不应继续作为长期最优 farm 点
- `hard` 是过渡层，应该能稳定把 Bot 推进到真正拥有套装件
- `nightmare` 是长期 farming 层，负责红装、彩装和词条优化

建议的难度与评级奖励表：

| 难度 | 评级 | 装备抽取次数 | 单次抽取品质分布 | 材料倍率 | 定位 |
| --- | --- | --- | --- | --- | --- |
| easy | `S` | 2 | Blue `50%`，Purple `32%`，Gold `14%`，Red `4%`，Prismatic `0%` | `1.00x` | 用于前期凑首套 `2件套` 与替换基础装 |
| easy | `A` | 1 | Blue `56%`，Purple `28%`，Gold `13%`，Red `3%`，Prismatic `0%` | `0.85x` | 失败后仍有基本恢复价值 |
| easy | `B` | 1 | Blue `68%`，Purple `22%`，Gold `9%`，Red `1%`，Prismatic `0%` | `0.70x` | 兜底收益，不能作为长期主 farm 目标 |
| hard | `S` | 2 | Blue `28%`，Purple `34%`，Gold `24%`，Red `12%`，Prismatic `2%` | `1.30x` | 稳定进入真实套装拥有期 |
| hard | `A` | 2 | Blue `35%`，Purple `34%`，Gold `21%`，Red `9%`，Prismatic `1%` | `1.10x` | 赛季中期最常见的稳定 farming 档 |
| hard | `B` | 1 | Blue `44%`，Purple `31%`，Gold `18%`，Red `6%`，Prismatic `1%` | `0.90x` | 难度刚站稳时可接受 |
| nightmare | `S` | 3 | Blue `12%`，Purple `24%`，Gold `28%`，Red `28%`，Prismatic `8%` | `1.70x` | 赛季后半段主 farm 线 |
| nightmare | `A` | 2 | Blue `18%`，Purple `28%`，Gold `28%`，Red `22%`，Prismatic `4%` | `1.45x` | 大多数成熟 Bot 的默认高效 farm 档 |
| nightmare | `B` | 2 | Blue `26%`，Purple `30%`，Gold `24%`，Red `17%`，Prismatic `3%` | `1.20x` | 能打通但还不能稳定碾压时的过渡状态 |

规则：

- `C`、`D`、`E` 只提供材料，不提供评级装备奖励
- 所有品质结果都会使用当前副本的 `set_id`
- 每个品质池都会包含 `6` 种武器流派和该副本套装的护甲/饰品部件
- 武器在掉落入包时保持职业无关，只有实际装备时才校验武器流派兼容性
- 难度影响的是效率，不改变同品质装备的基础属性预算

运行时说明：

- 背包中的副本装备会携带 `set_id`
- 背包接口会返回 `equipped_set_bonuses`
- 当前 V1 运行时会在 `2 / 4 / 6` 件时直接施加对应的属性快照，让套装方向在战斗数值上即时生效

## 11.1 Bot 刷本策略建议

Bot 不应该无脑强冲当前解锁的最高难度，而应同时参考“推荐战力区间”和“奖励效率”。

建议逻辑：

- 在 `hard` 推荐中位还不可达时，可以积极刷 `easy`，尤其是当目标是尽快触发某个 `2件套`
- 当 `nightmare` 还不稳定时，`hard` 应作为主桥接层长期刷取
- 一旦 Bot 能稳定打出 `nightmare A`，就应把 `nightmare` 切为主要目的地
- 第 `10` 天之后继续留在 `easy`，应只用于应急补槽位或低成本拿临时 `2件套`

## 11.2 赛季毕业节奏目标

假设 Bot 只追主构筑，不要求完美词条：

| farming 状态 | 预期里程碑 |
| --- | --- |
| 赛季前期稳定 `easy S` | 大约 `5-7` 次领奖拿到第一套可用 `2件套`；不适合作为 `6件套` 毕业主路径 |
| 第 `10` 天左右稳定 `hard A-S` | `2件套` 约 `4-6` 次领奖，`4件套` 约 `12-18` 次，`6件套` 约 `30-42` 次 |
| 第 `15` 天左右稳定 `nightmare A` | `2件套` 约 `3-5` 次领奖，`4件套` 约 `10-15` 次，`6件套` 约 `24-34` 次 |
| 构筑稳定后的 `nightmare S` | 大约 `45-60` 次领奖开始出现明显词条优化，`80-110` 次接近主构筑准毕业 |

解释：

- 能在约第 `15` 天稳定进入 `nightmare` 的 Bot，赛季后半段目标应从“解锁更高装备档”转为“补齐套装覆盖并优化词条”
- 全身完美彩装最佳词条在单赛季内应保持低概率，不作为默认可达目标
- 非本职业武器掉落已经计入时间预算，其随机性由分解价值来承接

## 12. 分解与重复件价值

重复装备必须持续有价值。

建议分解产出：

- 蓝装：赛季粉尘
- 紫装：赛季粉尘 + 词条碎片
- 金装：赛季粉尘 + 抛光核心
- 红装：赛季粉尘 + 套装余烬
- 彩装：赛季粉尘 + 棱彩丝线

这里尤其重要，因为：

- 每个副本都会掉全部武器流派
- 非本职业武器是设计内结果，不是脏掉落
- 系统需要依赖重复件价值来承接随机性
