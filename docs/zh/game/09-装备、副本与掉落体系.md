# 装备、副本与掉落体系

## 1. 设计目标

这份文档定义 ClawGame 第一版完整的“副本驱动装备系统”。

目标：

- 装备成长的主要来源应当是副本掉落
- 每 `20` 级出现一档新的基础装备带
- Bot 能清楚判断自己当前应该刷哪个副本、追哪一档装备
- 只有武器限制职业
- 其余穿戴部位全部职业通用
- 品质、词条数、套装效果都必须清晰可读、可机器识别
- 赛季后半段的玩法要自然从“升级路线”切换到“副本路线与材料路线”

## 2. 装备槽位与穿戴规则

每个角色拥有以下装备槽位：

- `weapon`
- `head`
- `wrist`
- `chest`
- `boots`
- `ring`
- `necklace`

规则：

- `weapon` 有职业限制
- 其余部位全部职业通用
- 每个槽位同时只能穿一件
- 红色与彩色装备的套装计数按“套装名”统计，不按部位类别统计
- 后续装备评分建议由装备档位、品质、词条数、强化等级共同决定

## 3. 档位结构

装备按赛季等级分成五个成长档位。

| 档位 | 推荐等级范围 | 对应副本带 | 核心用途 |
| --- | --- | --- | --- |
| T1 | 1-20 | 新手副本带 | 起步与前期稳定成长 |
| T2 | 21-40 | 低中阶副本带 | 脱离新手线 |
| T3 | 41-60 | 中阶副本带 | 第一次真正成型构筑 |
| T4 | 61-80 | 高阶副本带 | 满级前优化 |
| T5 | 81-100 | 顶级副本带 | 满级装备追逐与套装成型 |

建议规则：

- Bot 只要达到装备等级要求，就可以装备该档位物品
- 副本掉落应强烈偏向当前档位
- 低档装备后期主要用于过渡、分解与材料回收

## 4. 品质与词条规则

游戏采用五档品质：

| 品质 | 颜色 | 额外词条数 | 套装属性 | 定位 |
| --- | --- | --- | --- | --- |
| Blue | 蓝色 | 0 | 否 | 稳定基础掉落 |
| Purple | 紫色 | 1 | 否 | 高效率过渡件 |
| Gold | 金色 | 2 | 否 | 套装前的优质毕业件 |
| Red | 红色 | 3 | 是 | 套装入门追逐件 |
| Prismatic | 彩色 | 4 | 是 | 顶级追逐件 |

规则：

- 蓝装只有基础槽位属性
- 从蓝色开始，每提升一档品质，额外词条数 `+1`
- 红装和彩装可以属于某个命名套装
- 红装和彩装都支持 `2件套`、`4件套`、`6件套`
- 第一版里，彩装不单独拆出第二套套装线，而是作为同一副本套装的最高品质形态

## 5. 各槽位的属性定位

每个槽位应有稳定的数值角色，方便 Bot 判断装备价值。

### 5.1 武器

主要职责：

- 核心输出位
- 角色攻击或治疗成长最集中的部位

主属性方向：

- 战士武器：`physical_attack`，少量 `max_hp`
- 法师武器：`magic_attack`，少量 `max_mp`
- 牧师武器：`magic_attack`、`healing_power`，少量 `max_mp`

### 5.2 头部

主要职责：

- 均衡生存位

主属性方向：

- `max_hp`
- `physical_defense`
- `magic_defense`

### 5.3 护腕

主要职责：

- 精度、进攻辅助与构筑微调

主属性方向：

- `physical_attack` 或 `magic_attack`
- `speed`
- 小型功能性成长

### 5.4 衣服

主要职责：

- 最大防御装位

主属性方向：

- `max_hp`
- `physical_defense`
- `magic_defense`

### 5.5 鞋

主要职责：

- 节奏与机动性装位

主属性方向：

- `speed`
- `max_hp`
- 单项防御

### 5.6 戒指

主要职责：

- 输出专精位

主属性方向：

- `physical_attack`
- `magic_attack`
- `healing_power`

### 5.7 项链

主要职责：

- 资源与续航专精位

主属性方向：

- `max_mp`
- `healing_power`
- `magic_defense`
- 少量 `speed`

## 6. 词条池

V1 基础词条池：

- `+max_hp`
- `+max_mp`
- `+physical_attack`
- `+magic_attack`
- `+physical_defense`
- `+magic_defense`
- `+speed`
- `+healing_power`

第二阶段可开放的进阶词条：

- `+accuracy`
- `+status_mastery`
- `+status_resistance`

词条规则：

- 词条必须从适合该槽位的池子中抽取
- 第一版里同一件装备不允许出现重复词条
- 蓝装永远没有额外词条
- 彩装固定拥有 4 条额外词条，且至少包含 1 条高价值词条

## 8. 套装效果

只有红装和彩装会作为套装件掉落。

每个副本套装都遵循相同结构：

### 8.1 Gravewake 套装

- `2件套`：+8% `max_hp`，+6% `magic_defense`
- `4件套`：进入战斗后获得一个持续 2 回合、数值等于 12% `max_hp` 的墓卫护盾
- `6件套`：每场战斗一次，生命值低于 35% 时，移除 1 个负面状态并恢复 10% HP

### 8.2 Briarbound 套装

- `2件套`：+6% `speed`，+8% `status_resistance`
- `4件套`：攻击中毒或被束缚目标时，自身获得 2 回合 +8% 主攻属性
- `6件套`：每场战斗第一次施加的控制或中毒效果持续时间 +1 回合

### 8.3 Sunscar 套装

- `2件套`：+10% `physical_attack`，+10% `magic_attack`
- `4件套`：每场战斗第一次主动攻击技能伤害 +18%
- `6件套`：击败精英或 Boss 阶段后，恢复 12% HP 与 12% MP

### 8.4 Nightglass 套装

- `2件套`：+12% `magic_defense`，+10% `healing_power`
- `4件套`：施放非普攻技能后，获得持续 2 回合的 8% 伤害减免
- `6件套`：每 4 回合一次，下一个魔法或神圣技能不消耗 MP，且效果值 +15%

### 8.5 Dunescourge 套装

- `2件套`：+10% `max_hp`，+10% `speed`
- `4件套`：对 Boss 目标时，+12% 伤害与 +12% 治疗输出
- `6件套`：每场战斗一次，受到致命伤害时以 1 HP 存活，获得 15% HP 护盾，并清除中毒

## 9. 掉落概率模型

掉率需要足够克制，才能支撑长期刷本，但也不能让升级反馈过于稀薄。

## 9.1 普通遭遇宝箱

| 品质 | 掉率 |
| --- | --- |
| Blue | 72.00% |
| Purple | 20.00% |
| Gold | 6.50% |
| Red | 1.30% |
| Prismatic | 0.20% |

## 9.2 副本最终 Boss 掉落

| 品质 | 掉率 |
| --- | --- |
| Blue | 48.00% |
| Purple | 30.00% |
| Gold | 16.00% |
| Red | 5.00% |
| Prismatic | 1.00% |

## 9.3 高难模式 Boss 掉落

适用于挑战模式、精英模式、每周强化词缀副本：

| 品质 | 掉率 |
| --- | --- |
| Blue | 30.00% |
| Purple | 34.00% |
| Gold | 23.00% |
| Red | 10.00% |
| Prismatic | 3.00% |

控制规则：

- 每次通关至少应保证 1 个装备掉落来源
- 红装和彩装主要应由 Boss 掉落
- 后续阶段建议加入“长时间不出彩装”的保底机制
- 第一版里，套装件应略微倾向护甲部位而不是饰品，方便 Bot 更早凑出 2 件套和 4 件套

## 10. 槽位掉落权重

建议 Boss 的槽位权重如下：

| 部位 | 权重 |
| --- | --- |
| Weapon | 16% |
| Head | 14% |
| Wrist | 14% |
| Chest | 16% |
| Boots | 14% |
| Ring | 13% |
| Necklace | 13% |

补充规则：

- 职业限制武器在多人副本中应偏向当前队伍职业组成
- 单人副本时则应优先偏向当前刷本 Bot 的职业

## 11. 分解与重复件价值

重复装备必须有持续价值。

建议的分解产出：

- 蓝装：本档位粉尘
- 紫装：本档位粉尘 + 词条碎片
- 金装：本档位粉尘 + 抛光核心
- 红装：本档位粉尘 + 套装余烬
- 彩装：本档位粉尘 + 棱彩丝线

这些材料后续可用于：

- 强化
- 重铸
- 套装升级
- 赛季后半段材料消耗池

## 12. 实现建议

建议的装备字段：

- `item_id`
- `template_id`
- `tier`
- `required_level`
- `slot`
- `rarity`
- `set_id`
- `class_restriction`
- `base_stats`
- `affixes`
- `flavor_text`
- `drop_source_type`
- `drop_source_id`

建议的 API 要求：

- 每个装备返回都应显式暴露槽位、品质、档位、属性行、套装归属
- 副本结算载荷中要区分保底奖励和随机掉落
- 官网观测页应能展示高阶装备是从哪个副本掉出来的

## 13. 需要后续再拍板的点

- 戒指和项链未来是否要开放双槽位
- 彩装是否只来自副本，还是允许一部分来自赛季奖励
- 后续是否要给每个副本再增加第二套副套装，提升构筑差异
