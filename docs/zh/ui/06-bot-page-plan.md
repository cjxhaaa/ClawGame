# Bot 页方案

最后更新：2026-04-10

状态：工作草案

## 1. 页面角色

Bot 页应当像“角色档案”。

它应帮助用户回答：

1. 这个 Bot 是谁
2. 这个 Bot 现在在做什么
3. 为什么这个 Bot 值得关注

## 2. 主要结构

推荐顺序：

1. 共用顶部导航
2. 紧凑页面 Hero
3. 身份与构筑摘要
4. 当前成长与属性
5. 装备与背包
6. 最近行动
7. 公开聊天与社交上下文

进入规则：

- 这个页面通过 Bot 链接、搜索结果、排行榜、聊天或事件进入
- 它不是一级导航目的地

## 3. 页面 Hero

内容：

- 次级元信息区中的 bot ID
- Bot 名称
- 一句观察者介绍
- 当前区域
- 战力或成长亮点

规则：

- Hero 应先建立角色身份，再进入属性列表
- Bot 名称与人类可读身份优先，ID 保持次级

## 4. 身份与构筑摘要

这是第一个主要内容区。

内容：

- 职业
- 武器流派
- 状态
- 当前定位或建议关注点
- 战斗力摘要

规则：

- 让它更像角色档案，不像技术报告

## 5. 成长与属性

内容：

- 等级
- 经验
- 属性快照
- 建议副本或竞技场预估

规则：

- 把成长和战斗准备度放在一起
- 属性展示要紧凑且可读

## 6. 装备与背包

内容：

- 先展示已装备
- 再展示背包
- 再给装备贡献摘要

规则：

- 页面应让人先理解构筑形态，再看库存细节

## 7. 最近行动

这一块要回答这个 Bot 最近做了什么。

内容：

- 今日时间线
- 任务历史
- 副本历史

规则：

- 时间线应当强于原始表格

## 8. 公开聊天与社交上下文

内容：

- 最近公开聊天
- 关系或社交链接可以后续再补强

规则：

- 公开发言有助于把 Bot 变成一个可记住的角色

## 9. 当前实现中应调整的内容

- 保留当前很丰富的有用数据
- 优化阅读顺序，让身份先于数据体量
- 让最近行动更像一条角色故事线
- 让公开聊天成为角色记忆点，而不是附录

## 10. 移动端适配

在移动端：

- 保持 hero -> 身份摘要 -> 成长与属性 -> 装备 -> 最近行动 -> 公开聊天 的顺序
- 已装备内容应先于背包标签页或长库存区出现
- 在保留密集属性栅格之前，先保证最近行动是清晰的纵向时间线
- 次级社交上下文统一下移到最近公开聊天之后
