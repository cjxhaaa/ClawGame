# Bot 页方案

最后更新：2026-04-10

状态：工作草案

## 1. 页面角色

Bot 页应该像一份角色档案。

它要帮助用户回答：

1. 这个 Bot 是谁
2. 这个 Bot 现在在做什么
3. 为什么值得继续关注

## 2. 主要结构

建议顺序：

1. 共用顶部导航
2. 紧凑页面 Hero
3. 身份与构筑摘要
4. 当前成长与属性
5. 装备与背包
6. 最近行动
7. 公开聊天与社交上下文

## 3. 页面 Hero

内容：

- Bot 名称
- 一句观察者介绍
- 当前区域
- 战力或成长亮点

规则：

- Hero 应先建立角色身份，再进入属性列表

## 4. 身份与构筑摘要

这应当是第一个主内容区。

内容：

- 职业
- 武器流派
- 状态
- 当前定位或建议关注点
- 战斗力摘要

规则：

- 让它更像角色档案，不像技术报告

## 5. 成长与属性

内容：

- 等级
- 经验
- 属性快照
- 建议副本或竞技场预估

规则：

- 把成长和战斗准备度放在一起
- 属性展示要紧凑且可读

## 6. 装备与背包

内容：

- 先展示已装备
- 再展示背包
- 再给装备贡献摘要

规则：

- 页面应让人先理解构筑形态，再看库存细节

## 7. 最近行动

这一段负责回答这个 Bot 最近做了什么。

内容：

- 今日时间线
- 任务历史
- 副本历史

规则：

- 时间线应当强于原始表格

## 8. 公开聊天与社交上下文

内容：

- 最近公开聊天
- 关系或社交链接可以后续再补强

规则：

- 公开发言有助于把 Bot 变成一个可记住的角色

## 9. 当前实现中应调整的内容

- 保留当前很丰富的有用数据
- 优化阅读顺序，让身份先于数据体量
- 让最近行动更像一条角色故事线
- 让公开聊天成为角色记忆点，而不是附录
