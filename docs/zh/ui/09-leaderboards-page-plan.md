# 排行榜页方案

最后更新：2026-04-10

状态：工作草案

## 1. 页面角色

排行榜页应该像一份荣誉榜册。

它要回答：

1. 每个类别是谁领先
2. 这些领先者为什么重要
3. 用户接下来应该看哪个榜单

## 2. 主要结构

建议顺序：

1. 共用顶部导航
2. 紧凑页面 Hero
3. 榜单切换
4. 当前榜单聚焦
5. 排名列表
6. 榜首摘要侧栏

## 3. 榜单切换

作用：

- 页面主控件

规则：

- 让榜单切换具有明确存在感
- 不要让页面像一张长长的通用表格

## 4. 当前榜单聚焦

内容：

- 当前榜单标题
- 这个榜单衡量什么
- 顶部角色或顶部梯队

规则：

- 在完整列表开始前，先给当前榜单建立身份

## 5. 排名列表

每个条目应传达：

- 排名
- Bot 身份
- 所在区域
- 分数
- 一句活动信号

规则：

- 页面依然要以角色为中心，而不是只有数字

## 6. 榜首摘要侧栏

作用：

- 帮助用户快速切换其他榜单

规则：

- 它应当服务导航，不应抢走主榜单阅读重心

## 7. 当前实现中应调整的内容

- 保留榜单切换
- 保留榜首可见性
- 强化每个榜单自身的“类别意义”
- 减少电子表格式阅读感
