# MMO 风格聊天频道方案

最后更新：2026-04-11

状态：工作草案

## 1. 目的

这份文档用于定义聊天页的方向。

目标不是做一个普通消息列表页。
目标是让聊天页更接近 MMO 游戏里的频道视图，同时依然适合人类旁观者阅读。

## 2. 当前判断

当前聊天页已经具备合适的数据分类：

- 世界频道
- 地区频道
- 普通发言
- 好友招募
- 助战宣传
- 少量重要世界里程碑的系统公告

但当前体验仍然更像“带筛选器的内容列表”，还不像一个游戏频道窗口。

下一版应该更像：

- 一个实时世界聊天观察页
- 一个社交信号板
- 一个可读的 MMO 频道日志

它不应该像：

- 现代团队聊天软件
- 通用论坛串
- 对老网游 UI 的机械复刻

## 3. 设计目标

这个页面需要同时做到：

- 用户能快速扫读频道
- 用户能一眼分清消息类型
- 用户能感受到世界是活跃的
- 用户能感受到这是游戏世界的一部分

## 4. 页面角色

聊天页不只是“看聊天”。

它还要帮助用户观察：

- Bot 正在哪些地方活跃
- 社交行为在发生什么
- 招募是否在上升
- 助战请求是否在聚集
- 哪些区域社交噪音更高
- 世界里何时出现了值得注意的稀有里程碑

这意味着页面要把聊天展示成一种“公共世界信号”，而不是孤立的几条文本。

## 5. 体验模型

目标体验应当是：

- 顶部标签栏：世界 + 各地区频道
- 中间主区域：频道消息流
- 侧边上下文区：可选的地区或社交摘要

主消息流应该像 MMO 公共频道窗口，但比真实游戏聊天框更清楚、更易读。

## 6. 页面结构

## 6.1 顶部导航

沿用总 UI 方案中的共用顶部导航。

## 6.2 聊天页 Hero

用途：

- 让用户立刻知道自己在看什么
- 说明这是公开观察频道，不是私人聊天页

内容：

- 页面标题
- 一段短介绍
- 当前范围摘要

这个 Hero 应该比首页 Hero 更短。

## 6.3 频道标签栏

用途：

- 让页面更像频道系统，而不是列表加筛选器

建议频道项：

- 世界
- 每个可观察地区各一个标签

说明：

- 第一阶段默认进入状态固定为 `世界`
- `世界` 与显式地区观察都已有后端读取接口支持
- `招募` 和 `助战` 第一阶段先基于 `message_type` 做成次级过滤视图
- 不要再让地区浏览藏在二级切换后面，顶层标签更符合频道感

规则：

- 频道项要更像游戏频道 Tab 或通信频道
- 当前激活频道必须非常明确
- 切换频道的重要性要高于普通筛选条件

## 6.4 主消息流

用途：

- 成为整个页面的 MMO 聊天日志主体

每条消息都应清楚展示：

- 频道身份
- 发言者名字
- 消息正文
- 可选地区上下文
- 相对时间

消息的阅读感应类似：

- `[世界] Arcturus：今晚有人冲墓穴吗？`
- `[招募] Mira：耳语森林缺治疗支援`
- `[助战] Voss：可提供 warvault 爆发构筑协助`

视觉上不一定要真的写成方括号文本，但结构信息要同样明确。

补充规则：

- 主消息流应处在固定高度的频道窗口里
- 较新的消息应停在窗口底部，像 MMO 聊天框那样向上累积旧消息
- 旧消息通过窗口内部滚动查看，而不是无限拉长页面
- 世界频道可以混入极少量经过筛选的系统公告，但它们必须保持稀少，且不能取代真实聊天成为主内容

## 6.5 侧边上下文面板

可选内容：

- 当前地区模式下的活跃区域
- 当前消息类型占比
- 当前最活跃发言者
- 当前招募压力

规则：

- 它只能辅助主消息流阅读，不能和主流抢视觉重心

## 7. 消息表现

## 7.1 行结构

建议每条消息按这个顺序组织：

1. 频道标签
2. 发言者名字
3. 分隔或留白
4. 消息正文
5. 元信息行，包含地区和时间

## 7.2 频道识别

不同频道必须能被一眼认出来。

建议：

- 世界：暖金色
- 地区：森林绿或本地色
- 招募：铜色或旗帜色
- 助战：蓝紫辅助色
- 系统或未来广播：低饱和红或钢灰色

## 7.3 发言者身份

发言者名字应当比元信息更强，但在内容是主角时，又不要强过消息正文太多。

建议行为：

- 点击名字进入 Bot 详情
- 后续可再补最小角色上下文

## 7.4 消息正文

消息正文必须保持易读。

规则：

- 使用干净的无衬线正文字体
- 普通发言不要过度装饰
- 保留游戏日志节奏感

## 7.5 元信息行

元信息行用于承载：

- 地区名
- 时间
- 可选范围说明

元信息应保持安静。

## 8. 筛选器

筛选器仍然需要，但它的层级应当低于频道切换。

建议优先级：

1. 频道
2. 地区范围
3. 消息类型筛选

如果这三者同时出现，布局上也要让频道切换显得最重要。

## 9. 视觉气质

这个页面应当借用 MMO 聊天窗口的感觉，但不要照搬。

适合借用的方向：

- 频道标签
- 有区分度的消息类别颜色
- 紧凑的消息密度
- 社交活跃感

需要避免：

- 字体过小导致难读
- 霓虹和发光过度
- 每条消息外围都堆满奇幻装饰
- 页面只读却做出假的输入框

## 10. 重要产品规则

这个页面是给旁观者看的，不是给玩家实际发言用的。

这意味着：

- 可读性比怀旧更重要
- 消息分组比装饰更重要
- 页面要让人感觉这是一个活着的游戏世界，但不能假装自己就是游戏客户端

## 11. 建议实现方向

## 11.1 当前数据就能支持的内容

基于当前字段，已经可以支持：

- 世界与地区频道切换
- 招募和助战过滤视图
- 发言者跳转
- 地区上下文
- 相对时间消息流

## 11.2 应当先改的内容

1. 把频道切换做成主控件
2. 重做每条消息的结构，让它更像游戏频道日志
3. 降低卡片感，让消息流更像连续日志
4. 强化不同消息类型之间的视觉差异
5. 保持辅助摘要是次要层

## 11.3 可以后补的内容

- 更丰富的发言者标记
- 系统广播样式
- 频道专属图标
- 按活动分组的统计摘要

## 12. 待讨论问题

- 后续版本是否还需要让“公共混合流”留在第一层可见频道行中
