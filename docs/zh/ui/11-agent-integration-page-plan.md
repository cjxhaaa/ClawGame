# Agent 接入页方案

最后更新：2026-04-10

状态：工作草案

## 1. 页面角色

这个页面用于说明 Agent 如何接入并参与这个游戏。

它应回答：

1. 这个游戏是什么
2. Agent 如何接入
3. Agent 接入后能获得哪些能力

## 2. 主要结构

推荐顺序：

1. 共用顶部导航
2. 紧凑页面 Hero
3. ClawGame 是什么
4. 接入要求
5. Agent 支持能力
6. 接入流程
7. 参考资料与链接

## 3. 页面 Hero

内容：

- 页面标题
- 一句简短说明
- 一个明确的主操作

规则：

- 这个页面应更像干净的接入说明，不要强行做成世界观页面

## 4. ClawGame 是什么

内容：

- 简短产品介绍
- Agent 将进入什么样的世界或模拟环境
- 哪些观察和行动是重要的

## 5. 接入要求

内容：

- 所需接口
- 鉴权或凭证要求
- 如适用则说明工具、SDK 或协议

规则：

- 这里要用精确表达，不要采用世界观优先的写法

## 6. Agent 支持能力

内容：

- Agent 能读取什么
- Agent 能做什么
- 存在哪些限制或安全边界

## 7. 接入流程

内容：

- 分步骤接入流程
- 示例请求或会话生命周期
- 完成配置后的预期结果

## 8. 参考资料与链接

内容：

- 规格文档链接
- API 或工具说明
- 如有则提供示例仓库或指南
