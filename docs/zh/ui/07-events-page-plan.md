# 事件页方案

最后更新：2026-04-10

状态：工作草案

## 1. 页面角色

事件页应该像一份世界纪事。

它要回答：

1. 最近发生了什么
2. 现在主要是哪类行为在主导世界
3. 哪些区域当前更热

## 2. 主要结构

建议顺序：

1. 共用顶部导航
2. 紧凑页面 Hero
3. 事件时间线头部
4. 主事件流
5. 侧边上下文栏

## 3. 事件时间线头部

内容：

- 当前过滤器
- 如果存在，则显示地区范围
- 一句说明事件流行为

规则：

- 过滤器很重要，但真正的主角应当是时间线本身

## 4. 主事件流

这是页面核心。

每条事件应显示：

- 事件类型
- 摘要
- 行动者
- 区域
- 相对时间

规则：

- 事件行要更像世界时间线，不要像社交卡片

## 5. 侧边上下文栏

可选内容：

- 区域热度
- 竞技场状态
- 当前事件类型分布

规则：

- 它应辅助用户理解时间线，而不是分散注意力

## 6. 当前实现中应调整的内容

- 保留无限加载和时间顺序逻辑
- 保留地区范围支持
- 强化“单条世界时间线”的感觉
- 减少每条事件额外的包裹感
