# 竞技场页方案

最后更新：2026-04-10

状态：工作草案

## 1. 页面角色

竞技场页应该像一个赛事舞台。

它要回答：

1. 竞技场现在处于什么阶段
2. 当前最大的看点是什么
3. 哪些参赛者和对局值得关注

## 2. 主要结构

建议顺序：

1. 共用顶部导航
2. 紧凑页面 Hero
3. 本周竞技场简报
4. 最近竞技场信号
5. 积分赛或淘汰赛视图
6. 对局详情聚焦

## 3. 本周竞技场简报

内容：

- 当前阶段
- 下一次结算时间
- 场内人数
- 一句说明本周状态

规则：

- 要有戏剧感，但依然易读

## 4. 最近信号

用最近竞技场事件作为公共信号层。

规则：

- 保持简短
- 用来制造“正在发生”的感觉
- 不要代替竞技场主体结构

## 5. 积分赛视图

内容：

- 顶部焦点
- podium
- cutline 或重点门槛
- 易读的 ladder

规则：

- 要像竞赛榜，而不是电子表格

## 6. 淘汰赛视图

内容：

- 轮次进度
- 对局选择
- 若可用则展示已结算对局细节

规则：

- 让 bracket 更像一条舞台推进线

## 7. 当前实现中应调整的内容

- 保留当前强烈的阶段感
- 保留积分赛 / 淘汰赛拆分
- 降低重复重面板感
- 让顶部 spectacle 比下方细节更清楚
