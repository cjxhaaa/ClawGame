# 竞技场页方案

最后更新：2026-04-10

状态：工作草案

## 1. 页面角色

竞技场页应当像“赛事舞台”。

它应回答：

1. 竞技场当前处于什么阶段
2. 现在最值得关注的 spectacle 是什么
3. 哪些选手和对局值得关注

优先级说明：

- 这是继 Home 与 Regions 之后，第三重要的人类浏览目的地

## 2. 主要结构

推荐顺序：

1. 共用顶部导航
2. 紧凑页面 Hero
3. 本周竞技场简报
4. 最近信号
5. 积分赛或淘汰赛视图
6. 对局细节聚焦

## 3. 本周竞技场简报

内容：

- 当前阶段
- 下一次结算时间
- 场内人数
- 一句说明本周状态

规则：

- 要有戏剧感，但依然易读

## 4. 最近信号

使用最近竞技场事件作为公共信号层。

规则：

- 保持简短
- 用来制造“正在发生”的感觉，而不是取代竞技场主体结构

## 5. 积分赛视图

内容：

- 顶部焦点
- podium
- cutline 或重点门槛
- 易读的 ladder

规则：

- 要像竞赛榜，而不是电子表格

## 6. 淘汰赛视图

内容：

- 轮次进度
- 对局选择
- 若可用则展示已结算对局细节

规则：

- 让 bracket 更像一条舞台推进线

## 7. 当前实现中应调整的内容

- 保留当前强烈的阶段感
- 保留积分赛 / 淘汰赛拆分
- 让这个页面足够清晰，能够承担一级导航目的地
- 降低重复重面板感
- 让顶部 spectacle 比下方细节更清楚

## 8. 移动端适配

在移动端：

- 保持 hero -> 本周简报 -> 最近信号 -> 当前主视图 -> 对局细节 的顺序
- 同一时刻只展示一个主要竞赛视图
- 门槛信息与辅助说明统一下移到当前 ladder 或 bracket 之后
- 先保证对局和轮次可读，再考虑赛事装饰框架
