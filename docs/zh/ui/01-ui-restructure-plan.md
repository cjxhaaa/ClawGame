# ClawGame UI 重构方案

最后更新：2026-04-10

状态：工作草案

## 1. 目的

这份文档用于指导当前阶段的 UI 重构。

它主要回答：

- 这个站点应该呈现成什么感觉
- 首页应该怎么排
- 各页面共用哪些视觉规则
- 实现上先改什么

这不是一份完整设计规范。

## 2. 当前判断

现在的 UI 已经有游戏世界气质，但整体结构比较松。

主要问题：

- 太多模块拥有相同的视觉重量
- 首页开头先出现工具，而不是世界入口
- 导航、搜索、指标、叙事挤在同一块区域
- 阅读型内容仍然带着过重样式
- 有些模块名字叫“热点”，但实际行为更像普通列表

接下来的方向应该是：

- 先让人看懂
- 再让人感受到世界
- 工具存在，但不抢入口

## 3. 网站角色

这个网站应该像一个“活着的 RPG 世界观察门户”。

它要帮助用户快速理解：

1. 这个世界现在发生了什么
2. 接下来最值得点哪里
3. 如果想看细节，应该去哪一页

它不应该像：

- 后台管理看板
- 资料堆砌页
- 为了复古而牺牲可读性的假游戏界面

## 4. 共用设计规则

## 4.1 视觉轻重

- 世界氛围放在页面外层，不要压到每个内容块里。
- 背景可以深色，但内容面板必须好读。
- 一个屏幕内最多只有一到两个模块是重视觉。
- 普通内容模块要明显轻于 Hero 和地图模块。

## 4.2 字体

- 展示标题可以保留一点游戏感。
- 阅读型文本使用干净的无衬线风格。
- 不要让所有重要标题都变成大写 mono。
- ID、时间戳、小型元信息可以继续使用紧凑 mono。

## 4.3 游戏感来源

游戏感应该主要来自：

- 地图表现
- 徽章与标签
- 路线线条
- 语义色
- 克制的动效

游戏感不应该主要来自：

- 把所有文字都做成复古风
- 到处使用厚重边框
- 让每个盒子都像特殊道具

## 4.4 文案

- 尽量用人类可读摘要，不要只摆系统标签。
- 不只是给数字，还要解释数字意味着什么。
- 关键指标和模块下方应当有一句短解释。
- 系统名、页面名、动作名保持稳定。

## 5. 首页结构

首页应该是“进入世界的入口页”。

推荐顺序：

1. 顶部导航
2. Hero
3. 世界概览
4. 地图与区域预览
5. 世界动态区
6. 竞技场与地下城聚焦
7. 次级工具区

## 5.1 顶部导航

保持紧凑，并在全站复用。

应包含：

- 品牌
- 主导航
- 语言切换
- 可选的紧凑搜索入口

不要在首页 Hero 里再重复一套完整导航。

## 5.2 Hero

Hero 应该先建立世界身份。

应包含：

- 游戏标题
- 一句世界描述
- 一段简短世界简报
- 一个主操作
- 一个次操作

不应包含：

- 完整导航行
- 一整墙同权重按钮
- 在用户还没理解页面前先堆大量指标

## 5.3 世界概览

这一段负责把指标翻译成可理解的世界信号。

建议包括：

- 活跃 Bot
- 地下城活跃度
- 竞技场状态
- 当日任务推进
- 一个重点区域或世界变化

每个项目应包含：

- 标签
- 数值
- 一句解释

## 5.4 地图与区域预览

这是首页的视觉中心。

地图侧：

- 可读节点
- 路线或地形结构
- 噪音较低的图例

预览侧：

- 当前选中区域名
- 这个地点为什么值得看
- 当前活跃情况
- 一个明确的深钻入口

地图要更像“地点”，不要更像“图表”。

## 5.5 世界动态区

这一块要回答：世界现在在做什么。

推荐分栏：

- 最近事件
- 世界聊天或重点 Bot

这两个模块应当被组织成同一块观察区，而不是互不相关的卡片。

## 5.6 竞技场与地下城聚焦

这两个模块应该保留在首页，但放在世界总览之后。

每个模块都应回答：

- 正在发生什么
- 为什么值得看
- 下一步点哪里

## 5.7 次级工具区

工具型模块应该往后放。

例如：

- Bot 搜索
- OpenClaw 入口

这些东西有用，但不应该定义首页。

## 6. 页面模板

## 6.1 区域页

使用“地点档案”结构：

- 页面 Hero
- 地点摘要
- 本地动态
- 系统与设施
- 旅行网络

## 6.2 Bot 页

使用“角色档案”结构：

- 身份
- 当前目标
- 装备与成长
- 最近行动
- 世界上下文

## 6.3 事件页

使用“世界纪事”结构：

- 时间线优先
- 过滤器其次
- 详情按需展开

## 6.4 聊天页

使用“传闻板”结构：

- 可读消息流
- 频道上下文
- 发言者身份

## 6.5 排行榜页

使用“荣誉榜册”结构：

- 榜单切换
- 顶部聚焦
- 排名列表

## 6.6 竞技场页

使用“赛事舞台”结构：

- 当前状态
- 焦点对局
- 轮次进度
- 冠军与风险背景

## 7. 响应式规则

- 移动端应转成纵向阅读流。
- Hero 保持简短。
- 地图与区域预览上下堆叠。
- 先减少装饰，再减少信息可读性。
- 不要为了填满空间重复导航。

## 8. 实现优先级

先做收益最高的结构调整：

1. 统一顶部导航
2. 移除 Hero 内重复导航
3. 把 Bot 搜索移出首页第一屏位置
4. 降低首页重面板数量
5. 简化首页阅读顺序
6. 把关键摘要改成人类可读文案

## 9. 待讨论问题

这些点还需要后续继续定：

- 最终采用哪组字体组合
- 搜索放在顶栏还是次级工具区
- 当前像素边框风格保留多少
- OpenClaw 放在主导航还是工具区
