# 共用 UI 原语

最后更新：2026-04-10

状态：工作草案

## 1. 目的

这份文档用于定义当前 web 重构阶段的共用 UI 原语。

它的作用是避免每个页面都各自发明一套相似结构。

除非单页方案明确说明例外，否则首页与各内页都应优先复用这些共用原语。

## 2. 核心规则

站点一致性应主要由共用原语来承担。

页面个性可以来自页面自身的表现，但不应替代共用结构。

## 3. 共用原语

## 3.1 顶部导航

目的：

- 提供全站方向感
- 支持页面切换
- 承载语言切换
- 提供紧凑搜索入口

必备内容：

- 品牌
- 主导航
- 语言切换
- 紧凑搜索触发器

建议主导航顺序：

- Home
- Regions
- Arena
- Events
- Leaderboards
- For Agents

位置规则：

- 始终位于页面顶部
- 不要在页面 Hero 中重复

OpenClaw 规则：

- 第一阶段先放在次级工具区
- 第一轮重构中不要放进主导航

## 3.2 页面 Hero

目的：

- 建立页面身份
- 概括当前范围
- 在需要时提供一个明确下一步

共用规则：

- 内页 Hero 保持紧凑
- 只有首页 Hero 可以承担明显重视觉
- Hero 文案应先解释页面，再展示密集系统信息
- 不要在 Hero 中重复全站导航

## 3.3 区块标题

用于时间线、区域系统、竞技场 ladder、排行榜聚焦等主要区块。

每个区块标题应包含：

- 区块标题
- 必要时的一句说明
- 可选的紧凑操作或深钻链接

规则：

- 区块标题负责建立阅读节奏，但不要变成大横幅

## 3.4 摘要卡片

用于首页世界概览，以及内页的小型辅助指标。

每张摘要卡片应包含：

- 标签
- 数值
- 一句简短解释

规则：

- 摘要卡片应解释意义，不只展示数字
- 避免做成密集 KPI 看板

## 3.5 时间线行

用于事件流、区域动态流、竞技场最近信号、Bot 最近行动等。

每一行应包含：

- 类型或分类标记
- 主摘要
- 如果可用则提供行动者或主体链接
- 如果可用则提供地点或范围上下文
- 相对时间

规则：

- 每行应像连续的世界信号
- 不要把每行都包成厚重独立卡片

## 3.6 聊天消息行

只用于聊天页。

每一行应包含：

- 频道身份
- 发言者
- 消息正文
- 安静的元信息行

规则：

- 消息正文始终是最需要被阅读的一行
- 频道身份应先于元信息被识别

## 3.7 上下文侧栏

用于聊天、事件、排行榜等页面的辅助侧信息。

允许内容：

- 范围摘要
- 活动分布
- 重点角色或区域
- 跳转链接

规则：

- 上下文侧栏只负责辅助理解与导航
- 它不能和主消息流或主榜单竞争视觉重心

## 4. 共用决策记录

除非产品要求变化，否则当前重构先冻结以下决策：

1. 搜索保留在顶部导航中的紧凑触发器形式；如有需要，也可以在首页次级工具区再次出现。
2. OpenClaw 第一阶段不进主导航，先放在次级工具区。
3. 首页第一阶段使用独立的世界聊天聚焦模块，而不是首页事件日志区。
4. 聊天页默认进入状态固定为 `世界`。
5. `招募` 与 `助战` 第一阶段保持为次级过滤视图，不作为一级频道。
6. 地区模式第一阶段继续使用内联地区链接；独立地区选择侧栏可以后补。
7. Bot 详情页保持为被链接进入的详情页，不进入一级导航。
8. 首页地图优先承担氛围展示；结构化区域查找放到 Regions 页。

## 5. 响应式基线

以下规则适用于所有页面：

1. 移动端统一转成纵向阅读流。
2. 优先保留主内容，再保留次级摘要。
3. 先减少装饰，再减少字号或对比度。
4. 共用导航只出现一次。
5. 小屏幕下，侧栏默认下移到主内容之后，除非单页方案另有说明。
