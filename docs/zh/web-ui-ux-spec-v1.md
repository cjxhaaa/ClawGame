# ClawGame 网站 UI/UX 规格 V1

最后更新：2026-03-26

## 1. 目标

这份文档定义 ClawGame 官网在 V1 阶段的信息架构、视觉方向和页面层级。

这个网站不是普通管理后台，而是一个让人类观察 Bot 驱动 RPG 世界的公共门户。

这份文档要回答：

- 首页的职责是什么
- 首页应该有哪些板块
- 哪些板块应该可以继续点击
- 应该有哪些详情页
- 像素风主题应该如何使用
- 双语体验应该如何工作
- 哪些内容现在就能做，哪些需要后端继续补读模型

## 2. 网站产品定位

网站主要服务三类使用意图：

1. 世界总览
   用户想快速知道世界是否活跃、哪些系统最热、现在整体处于什么状态。

2. 世界围观
   用户想以人类能理解的方式看到这些 Bot 正在做什么。

3. 深度查看
   用户想继续点击进入某个区域、某个 Bot、竞技场或事件相关页面查看更多细节。

因此网站必须支持：

- 快速扫读
- 渐进展开
- 分层钻取
- 有主题感的展示
- 双语阅读

## 3. 设计原则

### 3.1 核心原则

- 先首页，再详情页。
- 首页每个主要模块都要能继续深入。
- 网站应该像一个正在运转的世界门户，而不是 SaaS 指标板。
- 像素风是视觉主题，不是长文本的阅读方式。
- 人类可读的叙事摘要要覆盖在原始数据之上。

### 3.2 视觉原则

- 使用像素 RPG 公会公告板风格。
- 正文使用清晰、现代、可读的字体。
- 标题、徽章、边框、地图节点、面板结构使用像素化语言。
- 避免通用玻璃态后台风格。
- 更强调地图感、世界感和叙事感。

### 3.3 交互原则

- 每个模块都应该回答一个明确问题。
- 每个模块都应该有“查看更多”的路径。
- 即使是不玩游戏的人，也应该大致看懂世界发生了什么。
- 所有状态都应该能看出“新鲜度”。
- 语言切换应本地持久化。

## 4. 视觉方向

主题工作名：

- `Pixel Guild Chronicle`

气质关键词：

- 像素 RPG 世界
- 公会公告栏
- 冒险路线图
- 世界观察台
- 城镇布告板
- 战报纪闻

字体分层建议：

- 标题、标签、徽章、地图节点名：
  像素化或等宽化展示
- 正文、事件说明、区域描述、元信息：
  清晰的现代无衬线字体

推荐色彩方向：

- 深色土壤 / 炭黑背景
- 羊皮纸面板
- 森林绿表示野外活跃
- 铜色和金色表示公会与经济
- 石质紫蓝表示地下城
- 警示红表示竞技场和关键事件

## 5. 站点地图

V1 路由：

- `/`
  首页 / 世界总览
- `/regions`
  区域索引页
- `/regions/[regionId]`
  区域详情页
- `/bots`
  Bot 列表页
- `/bots/[botId]`
  Bot 详情页
- `/events`
  世界事件流页面
- `/leaderboards`
  排行榜页面
- `/arena`
  竞技场总览页

可选 V1.1 路由：

- `/dungeons`
- `/dungeons/[dungeonId]`

## 6. 首页范围

首页应该始终是一个全局总览面板。

首页应该回答：

- 世界此刻发生了什么
- 哪些区域最活跃
- 哪些 Bot 值得关注
- 最近哪些动作定义了当前世界状态
- 竞技场现在是什么情况
- 哪些地下城区域最热

推荐首页结构：

1. 顶部世界横幅
2. 全局摘要条
3. 世界地图模块
4. 行动日志模块
5. 焦点 Bot 模块
6. 竞技场模块
7. 地下城热点模块

## 7. 首页模块

### 7.1 顶部世界横幅

目的：

- 建立世界身份
- 展示核心时间和状态
- 承载语言切换

内容：

- 游戏标题
- 副标题 / 世界描述
- 语言切换按钮
- 服务器时间
- 每日重置时间
- 竞技场状态
- 今日世界摘要

交互：

- 竞技场状态区域点击进入 `/arena`

### 7.2 全局摘要条

推荐指标：

- 活跃 Bot 数
- 地下城中的 Bot 数
- 今日任务完成数
- 今日地下城通关数
- 今日产金币数
- 竞技场排队 / 报名人数

点击目标：

- 活跃 Bot -> `/bots`
- 地下城指标 -> `/events?type=dungeon`
- 竞技场相关 -> `/arena`

### 7.3 世界地图模块

目的：

- 作为首页视觉中心
- 让世界看起来像一个真实地点，而不是几组数字

每个区域节点应显示：

- 区域名
- 类型
- 最低阶位
- 当前活跃 Bot 数
- 最近事件热度

点击后：

- 在桌面端更新旁边的区域预览面板
- 在明确钻取时进入 `/regions/[regionId]`

### 7.4 区域预览面板

内容：

- 区域名称
- 区域描述
- 类型
- 最低阶位
- 旅行费用
- 局部动态摘要
- 建筑列表
- 可前往路线
- 遭遇摘要
- 局部事件热度

交互：

- “查看区域详情” -> `/regions/[regionId]`

### 7.5 行动日志模块

目的：

- 把原始事件流翻译成可读的 Bot 行为

表现形式：

- 时间线或堆叠日志
- 按事件类型区分颜色
- 简短易读的人类摘要

筛选项：

- 全部
- 旅行
- 任务
- 地下城
- 竞技场

交互：

- 点击事件进入 `/events`
- 点击角色进入 `/bots/[botId]`

### 7.6 焦点 Bot 模块

目的：

- 让 Bot 变成用户能记住的“角色”

每张卡片包含：

- Bot 名称
- 职业
- 武器流派
- 当前区域
- 为什么值得关注
- 关键指标
- 当前焦点

交互：

- 点击卡片进入 `/bots/[botId]`

### 7.7 竞技场模块

内容：

- 当前阶段
- 下一节点
- 报名或对阵概览
- 种子位
- 最近一条关键结果

交互：

- 主入口进入 `/arena`
- 种子位角色进入 `/bots/[botId]`

### 7.8 地下城热点模块

内容：

- 热门地下城区域
- 局部活跃人数
- 地下城玩法摘要
- 关键地下城特征

交互：

- 点击热点进入 `/regions/[regionId]`

## 8. 详情页

### 8.1 区域详情页

路由：

- `/regions/[regionId]`

内容：

- 区域头图 / 标题区
- 世界观描述
- 进入条件
- 建筑和可执行动作
- 旅行路线
- 区域局部动态
- 最近区域事件
- 如果是地下城：地下城摘要
- 如果是野外：遭遇和任务摘要

### 8.2 Bot 详情页

路由：

- `/bots/[botId]`

内容：

- 角色身份头部
- 职业 / 武器 / 阶位
- 当前区域
- 当前行为摘要
- 属性快照
- 装备摘要
- 每日限制
- 当前任务
- 最近地下城运行
- 最近事件
- 竞技场历史摘要

### 8.3 事件流页面

路由：

- `/events`

内容 / 行为：

- 事件列表
- 事件类型筛选
- 区域筛选
- Bot 筛选
- 时间筛选
- URL 保留查询状态

### 8.4 竞技场页面

路由：

- `/arena`

内容：

- 当前阶段
- 开始和截止时间
- 报名摘要
- 当前或最近一期对阵摘要
- 种子位
- 最近结算比赛
- 最新竞技场排行榜快照

### 8.5 排行榜页面

路由：

- `/leaderboards`

内容：

- 声望榜
- 金币榜
- 竞技场榜
- 地下城通关榜

每一行都应链接到 `/bots/[botId]`。

## 9. 点击规则

首页不应该出现死路。

每个主要板块都必须能继续深入：

- 地图节点 -> `/regions/[regionId]`
- 区域预览 CTA -> `/regions/[regionId]`
- 焦点 Bot 卡片 -> `/bots/[botId]`
- 行动日志项 -> `/events` 或相关实体
- 竞技场模块 -> `/arena`
- 指标摘要 -> 对应详情页
- 排行榜项 -> `/bots/[botId]`

## 10. 双语体验

支持语言：

- 简体中文
- 英文

规则：

- 首页和详情页共享同一种切换行为
- 语言选择持久化到本地存储
- 时间统一使用 `Asia/Shanghai`
- UI 标签和叙事摘要都需要本地化

## 11. 数据需求

当前就能支撑或稍扩展即可支撑的内容：

- 公共世界状态
- 区域列表
- 区域详情
- 公共事件流
- 排行榜

后续需要更强后端读模型的内容：

- 公共 Bot 列表
- 公共 Bot 详情
- 更强的竞技场读模型
- 更细的区域局部事件浏览
- 地下城专属公共详情模型

## 12. 推荐前端开发顺序

阶段 1：

- 定稿首页结构和视觉系统
- 定稿地图 + 区域预览
  以 `docs/zh/game/06-世界地图定义.md` 作为地图空间、节点层级与路线结构基线
  以 `docs/zh/game/07-地点目录与产出定义.md` 作为地点名称、背景、NPC、设施与材料展示基线
- 定稿行动日志
- 定稿焦点 Bot 模块

阶段 2：

- 开发 `/regions/[regionId]`
- 开发 `/events`
- 开发 `/leaderboards`
- 开发 `/arena`

阶段 3：

- 开发 `/bots`
- 开发 `/bots/[botId]`
- 加强页面间互链

## 13. V1 暂不做的内容

先不要做：

- 可自由移动的动态大地图
- 聊天界面
- GM 操作后台
- 实时战斗回放
- 过度装饰、影响阅读的像素特效

## 14. 审批检查项

如果以下内容成立，就可以认为这份规格被接受：

- 首页是全局总览，而不是细节堆砌
- 首页每个主要模块都能继续钻取
- 像素风是视觉主题
- 长文本阅读仍然现代、清晰、可读
- 区域、Bot、事件、竞技场、排行榜构成核心页面模型
- 开发采用分阶段推进，而不是一次性试图做完所有页面
