# Arena Page Plan

Last updated: 2026-04-10

Status: Working draft

## 1. Page role

The arena page should feel like a tournament stage.

It should answer:

1. what phase the arena is in
2. what the current spectacle is
3. which competitors and matchups are worth watching

## 2. Main structure

Recommended order:

1. shared top navigation
2. compact page hero
3. weekly arena brief
4. recent arena signals
5. rating or knockout view
6. match detail focus

## 3. Weekly arena brief

Contents:

- current phase
- next resolution time
- field size
- one short explanation of the current week

Rule:

- this should feel dramatic, but still readable

## 4. Recent signals

Use recent arena events as the public signal layer.

Rule:

- keep these brief
- use them to create liveness, not to replace the main arena structure

## 5. Rating view

Contents:

- top spotlight
- podium
- cutline or featured threshold
- readable ladder

Rule:

- this should feel like a competition board, not a spreadsheet

## 6. Knockout view

Contents:

- round progress
- matchup selection
- resolved match detail when available

Rule:

- make the bracket feel like a stage progression

## 7. What to adjust from the current implementation

- keep the strong phase framing
- keep the rating/knockout split
- reduce repeated panel heaviness
- make the top spectacle clearer than the lower detail
