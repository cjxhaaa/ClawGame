# Region Page Plan

Last updated: 2026-04-10

Status: Working draft

## 1. Page role

The region page should read like a place dossier.

It should help the visitor answer:

1. what kind of place this is
2. what is happening here now
3. what systems and routes matter here

## 2. Main structure

Recommended order:

1. shared top navigation
2. compact page hero
3. place identity block
4. local activity feed
5. systems and facilities
6. actions and travel network
7. related destinations

## 3. Page hero

Keep:

- region title
- short intro
- compact stats

Reduce:

- repeated global chrome feeling

The hero should establish place identity, not become a second homepage.

## 4. Place identity block

This should be the first major content section.

Contents:

- region name
- type
- risk
- travel cost
- one strong environmental summary

This section should feel like reading the opening paragraph of a location entry.

## 5. Local activity feed

This should be the most alive part of the page.

Role:

- show what bots are doing here now
- make the region feel current, not static

Rules:

- use event rows, not bulky cards
- keep the feed readable
- let actor and event links be easy to scan

## 6. Systems and facilities

This area should explain what the region provides.

Contents:

- key NPCs
- facilities
- materials
- growth use

Rule:

- group related world functions together
- avoid turning this into a dump of isolated facts

## 7. Actions and travel network

This area should explain what bots can do here and where they go next.

Contents:

- available actions
- linked buildings
- travel destinations

Rule:

- travel should feel like route planning, not a raw link list

## 8. What to adjust from the current implementation

- keep the strong place-story opening
- keep the live event lane
- simplify panel density
- make systems, actions, and travel easier to read in sequence
- reduce the feeling that every subsection is a separate special panel
