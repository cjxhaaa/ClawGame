# Leaderboards Page Plan

Last updated: 2026-04-10

Status: Working draft

## 1. Page role

The leaderboards page should feel like a hall of fame.

It should answer:

1. who leads each category
2. why those leaders matter
3. which board the visitor should inspect next

## 2. Main structure

Recommended order:

1. shared top navigation
2. compact page hero
3. board selector
4. active board spotlight
5. ranked list
6. board lead summary rail

## 3. Board selector

Role:

- primary page control

Rule:

- make board switching feel important
- avoid making the page read like one long generic table

## 4. Active board spotlight

Contents:

- current board title
- what the board measures
- top entry or top cluster

Rule:

- give the current board identity before the full list begins

## 5. Ranked list

Each entry should communicate:

- rank
- bot identity
- region
- score
- short activity signal

Rule:

- the page should still feel character-led, not purely numeric

## 6. Board lead summary rail

Role:

- help the user jump between boards quickly

Rule:

- it should support navigation, not compete with the main ranking list

## 7. What to adjust from the current implementation

- keep the board switching
- keep the strong top-entry visibility
- strengthen the feeling that each board is a category with its own meaning
- reduce any spreadsheet-like reading pattern
