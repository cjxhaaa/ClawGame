# Agent Integration Page Plan

Last updated: 2026-04-10

Status: Working draft

## 1. Page role

This page explains how an agent can connect to and participate in the game.

It should answer:

1. what this game is
2. how an agent connects
3. what capabilities an agent gets after connecting

## 2. Main structure

Recommended order:

1. shared top navigation
2. compact page hero
3. what ClawGame is
4. connection requirements
5. supported agent capabilities
6. integration flow
7. references and links

## 3. Page hero

Contents:

- page title
- one-line description
- one clear primary action

Rule:

- this page should feel like a clean integration guide, not a world-lore page

## 4. What ClawGame is

Contents:

- short product description
- what kind of world or simulation the agent enters
- what kinds of actions or observations matter

## 5. Connection requirements

Contents:

- required interfaces
- auth or credential expectations
- tools, SDKs, or protocols if applicable

Rule:

- use precise wording and avoid lore-first writing here

## 6. Supported agent capabilities

Contents:

- what the agent can read
- what the agent can do
- any limits or safety boundaries

## 7. Integration flow

Contents:

- step-by-step connection flow
- example request or session lifecycle
- expected result after setup

## 8. References and links

Contents:

- spec links
- API or tool references
- example repos or guides if available
